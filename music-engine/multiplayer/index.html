Troubleshootingâ€‹

Problem: Music doesn't play on clientsâ€‹

Symptoms:

  • Server/host hears music
  • Clients hear nothing

Solutions:

1. Check Authorityâ€‹

// âŒ WRONG: Only runs on server

{
    PushMusicLayer(...);
}

// âœ… CORRECT: Runs on all clients

2. Use Multicast RPCâ€‹

UFUNCTION(NetMulticast, Reliable)
void Multicast_PlayMusic(FERP_MusicLayer Layer);

void AMyActor::TriggerMusic()
{
    if (HasAuthority())
    {
        Multicast_PlayMusic(MyLayer);
    }
}

void AMyActor::Multicast_PlayMusic_Implementation(FERP_MusicLayer Layer)
{
    UERP_MusicSubsystem* MusicSys = GetGameInstance()->GetSubsystem<UERP_MusicSubsystem>();
    if (MusicSys)
    {
        MusicSys->PushMusicLayer(Layer);
    }
}

Problem: Music volumes trigger for wrong playerâ€‹

Symptoms:

  • Player A enters volume
  • Player B's music changes instead

Solution:

This shouldn't happen - volumes use local overlap events. If it does:

  1. Check your Blueprint: Are you calling music functions on the overlapping actor instead of locally?
  2. Verify actor filter tags are set correctly
  3. Make sure the volume itself isn't replicated weirdly

Correct pattern:

Check: Is overlapping actor local player character?
  â†“

Advanced: Custom Replicationâ€‹

If you need server-authoritative music (rare), create a replicated actor:

UCLASS(Replicated)
class AReplicatedMusicController : public AActor
{
    GENERATED_BODY()

    // Replicated music state
    UPROPERTY(ReplicatedUsing=OnRep_CurrentMusic)
    FName CurrentMusicLayer;

    // Called on clients when music changes
    UFUNCTION()
    void OnRep_CurrentMusic()
    {
        UERP_MusicSubsystem* MusicSys = GetGameInstance()->GetSubsystem<UERP_MusicSubsystem>();
        if (MusicSys && !CurrentMusicLayer.IsNone())
        {
            // Apply replicated music state
            MusicSys->PushMusicLayerByName(CurrentMusicLayer);
        }
    }

    // Server function to change music for everyone
    UFUNCTION(BlueprintCallable, BlueprintAuthorityOnly)
    void ServerSetMusic(FName LayerName)
    {
        CurrentMusicLayer = LayerName;
        // Replication automatically updates all clients
    }
};

When to use this:

  • Rhythm games requiring perfect sync
  • Server-controlled boss music phases
  • Turn-based games with music tied to game state

When NOT to use:

  • 99% of normal games - default client-side is better
