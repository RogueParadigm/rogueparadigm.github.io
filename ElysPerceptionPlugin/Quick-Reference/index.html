Quick Reference

Quick reference and cheat sheet for ElysPerceptionPlugin.

Common Patterns​

Basic Targeting Setup​

1. Add ERPPerceptionComponent to PlayerController
2. Configure Target Pipeline:
   - Channel Id: "Target"
   - Sampler: ERPSphereOverlapSampler (range: 2000)
   - Filter: ERPTargetableFilter
   - Scorer: ERPDistanceScorer (normalize: true)
   - Resolver: ERPLowestScoreResolver
3. Add ERPChannelListenerComponent (Channel: "Target")
4. Bind to OnCandidateAcquired/Lost
5. Implement IERPTargetable on targets

Basic Interaction Setup​

1. Configure Interaction Pipeline (range: 300)
2. Add ERPInteractionComponent (Channel: "Interaction")
3. Bind to OnDescriptorChanged
4. Implement IERPInteractable on interactables
5. Handle Interact input

Component Quick Reference​

ERPPerceptionComponent​

// Get candidate for channel
AActor* Target = Perception->GetChannelCandidate("Target");

// Check if has candidate
if (Perception->HasChannelCandidate("Interaction"))
{
    // ...
}

// Bind to events
Perception->OnChannelCandidateAcquired.AddDynamic(this, &AMyPC::OnAcquired);

ERPChannelListenerComponent​

// Configure
TargetListener->ChannelId = "Target";

// Get current candidate
AActor* Target = TargetListener->GetCurrentCandidate();

// Bind to events (channel-specific)
TargetListener->OnCandidateAcquired.AddDynamic(this, &AMyPC::OnTargetAcquired);

ERPInteractionComponent​

// Configure
InteractionComp->ChannelId = "Interaction";

// Get current interactable
AActor* Interactable = InteractionComp->GetCurrentCandidate();

// Get descriptor
FERPInteractionDescriptor Desc;
if (InteractionComp->GetCurrentDescriptor(Desc))
{
    // Update UI with Desc.PromptText
}

// Bind to events
InteractionComp->OnDescriptorChanged.AddDynamic(this, &AMyPC::OnDescriptorChanged);

Interface Implementation​

IERPTargetable (Blueprint)​

Class Settings → Interfaces → Add ERPTargetable

Function: Can Be Targeted By (Targeting Actor)
  → Return: true

// Optional: Custom validation
Function: Can Be Targeted By (Targeting Actor)
  → Branch: Is Alive AND Not In Cover
    True: → Return true
    False: → Return false

IERPInteractable (Blueprint)​

Class Settings → Interfaces → Add ERPInteractable

Function: Can Be Interacted With (Interacting Actor)
  → Return: true

Function: Get Interaction Descriptor (Out Descriptor)
  → Make Interaction Descriptor
    - Display Name: "Door"
    - Prompt Text: "Press E to Open"
  → Set Out Descriptor

Function: Interact (Interacting Actor)
  → [Execute interaction logic]

IERPTargetable (C++)​

class AMyEnemy : public ACharacter, public IERPTargetable
{
    GENERATED_BODY()

public:
    virtual bool CanBeTargetedBy_Implementation(AActor* TargetingActor) const override
    {
        return bIsAlive && !bIsInCover;
    }
};

IERPInteractable (C++)​

class AMyDoor : public AActor, public IERPInteractable
{
    GENERATED_BODY()

public:
    virtual bool CanBeInteractedWith_Implementation(AActor* InteractingActor) const override
    {
        return true;
    }

    virtual void GetInteractionDescriptor_Implementation(FERPInteractionDescriptor& OutDescriptor) const override
    {
        OutDescriptor.DisplayName = TEXT("Door");
        OutDescriptor.PromptText = bIsOpen ? TEXT("Press E to Close") : TEXT("Press E to Open");
    }

    virtual void Interact_Implementation(AActor* InteractingActor) override
    {
        bIsOpen = !bIsOpen;
        // Animation, sound, etc.
    }
};

Pipeline Configuration​

Distance-Based Targeting​

Sampler: ERPSphereOverlapSampler (range: 3000)
Filter: ERPTargetableFilter
Scorer: ERPDistanceScorer (normalize: true)
Aggregator: ERPIdentityAggregator
Resolver: ERPLowestScoreResolver

Camera-Aware Targeting​

Sampler: ERPSphereOverlapSampler
Filters: ERPTargetableFilter
Scorers:
  - ERPDistanceScorer (normalize: true)
  - ERPConeScorer (cone: 45 degrees)
Aggregator: ERPAdditiveAggregator
Resolver: ERPLowestScoreResolver

Screen-Space Targeting​

Sampler: ERPSphereOverlapSampler
Filters: ERPTargetableFilter
Scorers:
  - ERPViewportEllipseScorer (radius: 200px)
Aggregator: ERPIdentityAggregator
Resolver: ERPLowestScoreResolver

Debugging​

Enable Debug Logging​

ERPPerceptionComponent details:
  - bDebugPerception: ✓

Console Commands​

showdebug PERCEPTION  // Show debug overlay
stat GAME            // Show performance stats

Debug Output​

When bDebugPerception enabled:

  • Logs winner candidate per channel each tick
  • Shows candidate count per channel
  • Displays score information

Performance Tips​

Reduce Sampling Range​

ERPPerceptionComponent:
  - DefaultSamplingRange: 1500 (instead of 3000)

Adjust Tick Rate​

// In constructor
PerceptionComponent->PrimaryComponentTick.TickInterval = 0.1f; // 10 Hz instead of 60 Hz

Optimize Sampler​

ERPSphereOverlapSampler:
  - ActorClassFilter: Specific class (not AActor)
  - TraceChannel: Specific channel

Use Filters Early​

Order filters by cost:
  1. ERPDistanceFilter (cheap range check)
  2. ERPTargetableFilter (interface check)
  3. Custom filters (more expensive logic)

Common Issues​

Target Not Detected​

Check:

  • Target implements IERPTargetable
  • CanBeTargetedBy returns true
  • Target within DefaultSamplingRange
  • Collision settings match TraceChannel

Events Not Firing​

Check:

  • Bound in BeginPlay (not constructor)
  • Used AddDynamic for delegates
  • ChannelId matches between pipeline and listener
  • Component is ticking

Performance Issues​

Fix:

  • Lower DefaultSamplingRange
  • Increase TickInterval
  • Tighten sampler filters
  • Use lightweight scorers

Score Convention​

Lower score = better (by convention)

Examples:

  • Distance: 0.0 (very close) to 1.0 (far)
  • Angle: 0.0 (center view) to 1.0 (edge)
  • Screen: 0.0 (center screen) to 1.0 (edge)

Ensure all custom scorers follow this convention for aggregator compatibility.

Multi-Channel Setup​

// Listen to multiple channels
Perception->OnChannelCandidateAcquired.AddDynamic(this, &AMyPC::OnAcquired);

void AMyPC::OnAcquired(FName ChannelId, AActor* Candidate)
{
    if (ChannelId == "Target")
    {
        // Handle targeting
    }
    else if (ChannelId == "Interaction")
    {
        // Handle interaction
    }
}

Or use multiple listeners:

// Target listener
TargetListener->ChannelId = "Target";
TargetListener->OnCandidateAcquired.AddDynamic(this, &AMyPC::OnTargetAcquired);

// Interaction listener
InteractionListener->ChannelId = "Interaction";
InteractionListener->OnCandidateAcquired.AddDynamic(this, &AMyPC::OnInteractionAcquired);
