Customization Guide

Create custom pipeline components and extend ElysPerceptionPlugin for advanced use cases.

When to Customize​

Create custom components when:

  • Default components don't fit your needs
  • You need game-specific logic (health priority, threat levels)
  • You want special filtering (line of sight, team-based)
  • You need unique scoring (custom gameplay metrics)

Extension Points​

The plugin provides interfaces for each pipeline stage:

  • IERPSampler - Custom candidate gathering
  • IERPFilter - Custom filtering logic
  • IERPScorer - Custom scoring logic
  • IERPAggregator - Custom score combination
  • IERPResolver - Custom winner selection

Blueprint vs C++​

BlueprintC++
Quick prototypingBest performance
Designer-friendlyType safety
No compilationFull engine access
Limited debuggingAdvanced debugging

Choose based on complexity and performance requirements.

Custom Samplers​

Purpose​

Samplers gather potential candidates. Customize when you need:

  • Different spatial queries (raycast, box overlap)
  • Non-spatial queries (GameState actor lists)
  • Cached/optimized candidate lists

Interface​

UFUNCTION(BlueprintNativeEvent, Category = "ElysPerception|Pipeline")
void Sample(const FERPPerceptionContext& Context, float Range, TArray<AActor*>& OutCandidates) const;

C++ Example: Line Trace Sampler​

Header:

#pragma once

#include "Pipeline/Samplers/ERPSamplerBase.h"
#include "ERPLineTraceSampler.generated.h"

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPLineTraceSampler : public UERPSamplerBase
{
    GENERATED_BODY()

public:
    virtual void Sample_Implementation(
        const FERPPerceptionContext& Context,
        float Range,
        TArray<AActor*>& OutCandidates) const override;

    UPROPERTY(EditAnywhere, Category = "Sampler")
    TEnumAsByte<ECollisionChannel> TraceChannel = ECC_Visibility;
};

Implementation:

#include "ERPLineTraceSampler.h"

void UERPLineTraceSampler::Sample_Implementation(
    const FERPPerceptionContext& Context,
    float Range,
    TArray<AActor*>& OutCandidates) const
{
    if (!Context.ContextActor) return;
    
    UWorld* World = Context.ContextActor->GetWorld();
    if (!World) return;
    
    FVector Start = Context.Origin;
    FVector End = Context.Origin + Context.Forward * Range;
    
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(Context.ContextActor);
    
    FHitResult HitResult;
    if (World->LineTraceSingleByChannel(HitResult, Start, End, TraceChannel, QueryParams))
    {
        if (AActor* HitActor = HitResult.GetActor())
        {
            OutCandidates.Add(HitActor);
        }
    }
}

Blueprint Example​

Create Blueprint class derived from ERPSamplerBase:

Function: Sample (Context, Range, OutCandidates)
  → Get All Actors of Class: EnemyClass
  → For Each Actor:
    → Get Distance (Context.Origin to Actor Location)
    → Branch: Distance < Range
      True: → Add to OutCandidates

Custom Filters​

Purpose​

Filters reject invalid candidates. Customize for:

  • Line of sight checks
  • State validation (alive, active)
  • Team/faction filtering
  • Permission checks

Interface​

UFUNCTION(BlueprintNativeEvent, Category = "ElysPerception|Pipeline")
bool Passes(const FERPPerceptionContext& Context, AActor* Candidate) const;

C++ Example: Line of Sight Filter​

Header:

#pragma once

#include "Pipeline/Filters/ERPFilterBase.h"
#include "ERPLineOfSightFilter.generated.h"

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPLineOfSightFilter : public UERPFilterBase
{
    GENERATED_BODY()

public:
    virtual bool Passes_Implementation(
        const FERPPerceptionContext& Context,
        AActor* Candidate) const override;

    UPROPERTY(EditAnywhere, Category = "Filter")
    TEnumAsByte<ECollisionChannel> TraceChannel = ECC_Visibility;
};

Implementation:

#include "ERPLineOfSightFilter.h"

bool UERPLineOfSightFilter::Passes_Implementation(
    const FERPPerceptionContext& Context,
    AActor* Candidate) const
{
    if (!Context.ContextActor || !Candidate) return false;
    
    UWorld* World = Context.ContextActor->GetWorld();
    if (!World) return false;
    
    FVector Start = Context.Origin;
    FVector End = Candidate->GetActorLocation();
    
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(Context.ContextActor);
    QueryParams.AddIgnoredActor(Candidate);
    
    FHitResult HitResult;
    bool bHit = World->LineTraceSingleByChannel(HitResult, Start, End, TraceChannel, QueryParams);
    
    return !bHit; // Pass if no obstruction
}

Blueprint Example: Team Filter​

Function: Passes (Context, Candidate)
  → Cast Context.ContextActor to MyCharacter
  → Cast Candidate to MyCharacter
  → Both Succeeded?
    True:
      → Get Team ID from both
      → Return: Teams are Different
    False:
      → Return: True

Custom Scorers​

Purpose​

Scorers assign numerical values. Customize for:

  • Game-specific metrics (health, threat)
  • Complex spatial scoring
  • Dynamic priorities

Interface​

UFUNCTION(BlueprintNativeEvent, Category = "ElysPerception|Pipeline")
float Score(const FERPPerceptionContext& Context, AActor* Candidate) const;

C++ Example: Health Priority Scorer​

Header:

#pragma once

#include "Pipeline/Scorers/ERPScorerBase.h"
#include "ERPHealthScorer.generated.h"

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPHealthScorer : public UERPScorerBase
{
    GENERATED_BODY()

public:
    virtual float Score_Implementation(
        const FERPPerceptionContext& Context,
        AActor* Candidate) const override;

    UPROPERTY(EditAnywhere, Category = "Scorer")
    bool bPrioritizeLowHealth = true;
};

Implementation:

#include "ERPHealthScorer.h"
#include "YourHealthComponent.h"

float UERPHealthScorer::Score_Implementation(
    const FERPPerceptionContext& Context,
    AActor* Candidate) const
{
    UYourHealthComponent* HealthComp = Candidate->FindComponentByClass<UYourHealthComponent>();
    if (!HealthComp) return 1.0f;
    
    float HealthPercent = HealthComp->GetHealthPercent();
    
    // Lower health = lower score (better)
    return bPrioritizeLowHealth ? HealthPercent : (1.0f - HealthPercent);
}

Blueprint Example: Threat Level Scorer​

Function: Score (Context, Candidate)
  → Cast Candidate to Enemy
  → Branch: Succeeded?
    True:
      → Get Threat Level (Enemy)
      → Return: Threat Level / 100.0 (normalized)
    False:
      → Return: 1.0

Custom Aggregators​

Purpose​

Aggregators combine multiple scores. Customize for:

  • Weighted averaging
  • Non-linear combinations
  • Min/max selection

Interface​

UFUNCTION(BlueprintNativeEvent, Category = "ElysPerception|Pipeline")
float Aggregate(const TArray<float>& Scores) const;

C++ Example: Weighted Average​

Header:

#pragma once

#include "Pipeline/Aggregators/ERPAggregatorBase.h"
#include "ERPWeightedAggregator.generated.h"

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPWeightedAggregator : public UERPAggregatorBase
{
    GENERATED_BODY()

public:
    virtual float Aggregate_Implementation(const TArray<float>& Scores) const override;

    UPROPERTY(EditAnywhere, Category = "Aggregator")
    TArray<float> Weights = {1.0f};
};

Implementation:

#include "ERPWeightedAggregator.h"

float UERPWeightedAggregator::Aggregate_Implementation(const TArray<float>& Scores) const
{
    if (Scores.Num() == 0) return 0.0f;
    
    float WeightedSum = 0.0f;
    float TotalWeight = 0.0f;
    
    for (int32 i = 0; i < Scores.Num(); ++i)
    {
        float Weight = (i < Weights.Num()) ? Weights[i] : 1.0f;
        WeightedSum += Scores[i] * Weight;
        TotalWeight += Weight;
    }
    
    return (TotalWeight > 0.0f) ? (WeightedSum / TotalWeight) : 0.0f;
}

Custom Resolvers​

Purpose​

Resolvers select the winner. Customize for:

  • Sticky targeting (prefer current)
  • Random selection
  • Priority tiers

Interface​

UFUNCTION(BlueprintNativeEvent, Category = "ElysPerception|Pipeline")
bool Resolve(const TArray<FERPPipelineScoredCandidate>& ScoredCandidates, FERPPipelineScoredCandidate& OutBest) const;

C++ Example: Sticky Target Resolver​

Header:

#pragma once

#include "Pipeline/Resolvers/ERPResolverBase.h"
#include "ERPStickyResolver.generated.h"

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPStickyResolver : public UERPResolverBase
{
    GENERATED_BODY()

public:
    virtual bool Resolve_Implementation(
        const TArray<FERPPipelineScoredCandidate>& ScoredCandidates,
        FERPPipelineScoredCandidate& OutBest) const override;

    UPROPERTY(EditAnywhere, Category = "Resolver")
    float StickyBias = 0.1f;

    UPROPERTY()
    mutable AActor* LastWinner = nullptr;
};

Implementation:

#include "ERPStickyResolver.h"

bool UERPStickyResolver::Resolve_Implementation(
    const TArray<FERPPipelineScoredCandidate>& ScoredCandidates,
    FERPPipelineScoredCandidate& OutBest) const
{
    if (ScoredCandidates.Num() == 0) return false;
    
    OutBest = ScoredCandidates[0];
    
    for (const FERPPipelineScoredCandidate& Candidate : ScoredCandidates)
    {
        float AdjustedScore = Candidate.Score;
        
        // Give current target score bonus
        if (Candidate.Actor == LastWinner)
            AdjustedScore -= StickyBias;
        
        if (AdjustedScore < OutBest.Score)
            OutBest = Candidate;
    }
    
    LastWinner = OutBest.Actor;
    return true;
}

Best Practices​

General Guidelines​

Keep Stages Pure:

  • Don't modify candidates or context
  • Only read and evaluate

Optimize for Performance:

  • Filters run per-candidate, keep fast
  • Cache expensive calculations
  • Avoid allocations in hot path

Normalize Scores:

  • Use 0.0-1.0 range when possible
  • Document expected ranges
  • Make combinable with other scorers

Handle Edge Cases:

  • Null checks
  • Empty arrays
  • Invalid context

Document Parameters:

  • Use UPROPERTY meta tags
  • Add tooltips
  • Provide sensible defaults
