Interaction Guide

Complete guide to implementing proximity-based interactions using ElysPerceptionPlugin.

Overview​

The interaction system builds on top of Perception to provide eligibility validation and state management. The plugin remains UI-agnostic and gameplay-agnostic.

Architecture:

  • Perception: Selects candidate locally (client-side detection)
  • Interaction: Validates eligibility and exposes state
  • Presentation: Adapts state to UI/FX (optional, your implementation)

ERPInteractionComponent​

ERPInteractionComponent is a framework-level helper providing complete interaction state management.

What it does:

  • Automatically subscribes to UERPPerceptionComponent channel events
  • Filters perception candidates using IERPInteractable interface
  • Validates using CanBeInteractedWith and optional UERPInteractionRule instances
  • Maintains current interaction state (candidate actor + descriptor)
  • Emits framework events: OnCandidateChanged, OnDescriptorChanged

Why it exists:

  • Perception = Detection (which actor is closest/best)
  • InteractionComponent = Eligibility (which actor can be interacted with)
  • Presentation/UI = External and optional (how to display)

This replaces the need for third-party interaction plugins.

Basic Setup​

1. Configure Interaction Pipeline​

On your PlayerController or Pawn, add and configure ERPPerceptionComponent:

Interaction Pipeline Settings:

  • Channel Id: Interaction
  • Default Sampling Range: 300.0 (shorter than targeting)
  • Sampler: ERPSphereOverlapSampler
    • Trace Channel: Visibility
    • Ignore Context Actor: ✓
  • Filters: ERPInteractableFilter
  • Scorers: ERPDistanceScorer (normalize by range)
  • Aggregator: ERPIdentityAggregator
  • Resolver: ERPLowestScoreResolver

2. Add Interaction Component​

Add ERPInteractionComponent to your PlayerController or Pawn:

Configuration:

  • Channel Id: Interaction (must match pipeline)

3. Implement Interactable Interface​

On actors you want to interact with (doors, items, NPCs):

Blueprint:

  1. Class Settings → Interfaces → Add ERPInteractable
  2. Implement Can Be Interacted With → return true
  3. Implement Get Interaction Descriptor:
Event: Get Interaction Descriptor
  → Make Interaction Descriptor
    - Display Name: "Door"
    - Prompt Text: "Press E to Open"
    - Interaction Tags: (empty)
    - Presentation Hints: (empty)
  → Set Out Descriptor

4. Bind to Events​

In your PlayerController:

Event BeginPlay
  → Get Component (InteractionComponent)
  → Bind Event to OnDescriptorChanged
    → Update UI with new descriptor

Event: Input Action Interact
  → Get Current Candidate from InteractionComponent
  → IsValid?
    True: → Execute interaction logic
    False: → (No interaction available)

Blueprint Example​

Complete Interaction System​

PlayerController Setup:

Components:

  • ERPPerceptionComponent (interaction pipeline configured)
  • ERPInteractionComponent (listening to "Interaction" channel)

Variables:

  • CurrentInteractable (Actor reference)
  • CurrentDescriptor (FERPInteractionDescriptor)

Event Graph:

// Handle descriptor changes
Event: OnDescriptorChanged (Candidate, Descriptor, ChannelId)
  → Set CurrentInteractable = Candidate
  → Set CurrentDescriptor = Descriptor
  → Update UI Widget with Descriptor
    - Set Prompt Text: Descriptor.PromptText
    - Set Display Name: Descriptor.DisplayName

// Execute interaction
Input Action: Interact
  → Get Current Candidate (InteractionComponent)
  → IsValid?
    True:
      → Cast to IERPInteractable
      → Execute Interact (on Interactable)
    False:
      → Print: "Nothing to interact with"

Interactable Actor (Door):

Variables:

  • bIsOpen (Boolean)

Event Graph:

Function: Can Be Interacted With (Interacting Actor)
  → Return: True

Function: Get Interaction Descriptor (Out Descriptor)
  → Branch: bIsOpen
    True:
      → Make Descriptor
        - Display Name: "Door"
        - Prompt Text: "Press E to Close"
    False:
      → Make Descriptor
        - Display Name: "Door"
        - Prompt Text: "Press E to Open"
  → Set Out Descriptor

Event: Interact (Interacting Actor)
  → Toggle bIsOpen
  → Play Door Animation

C++ Example​

PlayerController Setup​

YourPlayerController.h:

#pragma once

#include "CoreMinimal.h"
#include "GameFramework/PlayerController.h"
#include "Core/ERPPerceptionComponent.h"
#include "Interaction/ERPInteractionComponent.h"
#include "YourPlayerController.generated.h"

UCLASS()
class YOURPROJECT_API AYourPlayerController : public APlayerController
{
    GENERATED_BODY()

public:
    AYourPlayerController();

protected:
    virtual void BeginPlay() override;
    virtual void SetupInputComponent() override;

    UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
    UERPPerceptionComponent* PerceptionComponent;

    UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
    UERPInteractionComponent* InteractionComponent;

private:
    UFUNCTION()
    void OnDescriptorChanged(AActor* Candidate, const FERPInteractionDescriptor& Descriptor, FName ChannelId);

    void HandleInteractInput();
};

YourPlayerController.cpp:

#include "YourPlayerController.h"
#include "EnhancedInputComponent.h"

AYourPlayerController::AYourPlayerController()
{
    PerceptionComponent = CreateDefaultSubobject<UERPPerceptionComponent>(TEXT("Perception"));
    InteractionComponent = CreateDefaultSubobject<UERPInteractionComponent>(TEXT("Interaction"));
    InteractionComponent->ChannelId = FName("Interaction");
}

void AYourPlayerController::BeginPlay()
{
    Super::BeginPlay();
    
    InteractionComponent->OnDescriptorChanged.AddDynamic(this, &AYourPlayerController::OnDescriptorChanged);
}

void AYourPlayerController::SetupInputComponent()
{
    Super::SetupInputComponent();
    
    if (UEnhancedInputComponent* EIC = Cast<UEnhancedInputComponent>(InputComponent))
    {
        EIC->BindAction(InteractAction, ETriggerEvent::Triggered, this, &AYourPlayerController::HandleInteractInput);
    }
}

void AYourPlayerController::OnDescriptorChanged(AActor* Candidate, const FERPInteractionDescriptor& Descriptor, FName ChannelId)
{
    // Update UI with descriptor
    UE_LOG(LogTemp, Log, TEXT("Interaction: %s - %s"), *Descriptor.DisplayName, *Descriptor.PromptText);
}

void AYourPlayerController::HandleInteractInput()
{
    AActor* Interactable = InteractionComponent->GetCurrentCandidate();
    if (Interactable && Interactable->Implements<UERPInteractable>())
    {
        IERPInteractable::Execute_Interact(Interactable, GetPawn());
    }
}

Interactable Actor​

YourDoor.h:

#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "Core/ERPInteractable.h"
#include "YourDoor.generated.h"

UCLASS()
class YOURPROJECT_API AYourDoor : public AActor, public IERPInteractable
{
    GENERATED_BODY()

public:
    virtual bool CanBeInteractedWith_Implementation(AActor* InteractingActor) const override;
    virtual void GetInteractionDescriptor_Implementation(FERPInteractionDescriptor& OutDescriptor) const override;
    virtual void Interact_Implementation(AActor* InteractingActor) override;

protected:
    UPROPERTY(EditAnywhere, BlueprintReadWrite)
    bool bIsOpen = false;
};

YourDoor.cpp:

#include "YourDoor.h"

bool AYourDoor::CanBeInteractedWith_Implementation(AActor* InteractingActor) const
{
    return true;
}

void AYourDoor::GetInteractionDescriptor_Implementation(FERPInteractionDescriptor& OutDescriptor) const
{
    OutDescriptor.DisplayName = TEXT("Door");
    OutDescriptor.PromptText = bIsOpen ? TEXT("Press E to Close") : TEXT("Press E to Open");
}

void AYourDoor::Interact_Implementation(AActor* InteractingActor)
{
    bIsOpen = !bIsOpen;
    // Play animation, sound, etc.
}

Advanced Patterns​

Multiple Interaction Types​

Use InteractionTags in the descriptor to support multiple actions:

Interactable Actor:

Function: Get Interaction Descriptor
  → Branch: Is Chest Locked?
    True:
      → Make Descriptor
        - Display Name: "Locked Chest"
        - Prompt Text: "Press E to Lockpick"
        - Interaction Tags: Add "Lockpick"
    False:
      → Make Descriptor
        - Display Name: "Chest"
        - Prompt Text: "Press E to Open"
        - Interaction Tags: Add "Open"

Context-Sensitive Actions​

Pass player context to determine availability:

bool AYourDoor::CanBeInteractedWith_Implementation(AActor* InteractingActor) const
{
    // Only allow if player has key
    AMyPlayer* Player = Cast<AMyPlayer>(InteractingActor);
    return Player && Player->HasKey();
}

Dynamic Descriptors​

Descriptors are rebuilt on demand, so state changes are automatically reflected:

void AYourDoor::GetInteractionDescriptor_Implementation(FERPInteractionDescriptor& OutDescriptor) const
{
    OutDescriptor.DisplayName = TEXT("Door");
    OutDescriptor.PromptText = bIsOpen ? TEXT("Press E to Close") : TEXT("Press E to Open");
    // Automatically updates when bIsOpen changes
}

Multiplayer Considerations​

Perception and Interaction are client-side by default.

For multiplayer validation:

  1. Client: Detects and validates interaction eligibility locally
  2. Client → Server: Send RPC to request interaction
  3. Server: Validate distance, line of sight, permissions
  4. Server: Execute gameplay logic
  5. Server → Clients: Replicate state changes

The plugin does NOT handle server validation. Implement your own RPC validation.
