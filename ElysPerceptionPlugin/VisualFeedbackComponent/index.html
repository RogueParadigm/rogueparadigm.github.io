ERPBaseInteractionFeedbackComponent Guide

Complete guide for using the optional visual feedback component for interactions.

Overview​

UERPBaseInteractionFeedbackComponent is an optional helper that provides default visual feedback behavior for interactable actors.

What It Does​

  • Manages outline/highlight visibility
  • Shows/hides world-space interaction widgets
  • Forwards descriptor updates to your custom widgets
  • Provides Blueprint hooks for custom behavior

What It Does NOT Do​

  • Does NOT detect or validate interactions
  • Does NOT subscribe to events automatically
  • Does NOT execute gameplay logic
  • Does NOT require any specific UI structure

When to Use It​

? Use when:

  • You want quick, default visual feedback (outline + widget)
  • You want to avoid writing boilerplate glue code
  • You want a clean separation between logic and presentation

? Don't use when:

  • You have completely custom visual feedback needs
  • You prefer handling everything directly in PlayerController
  • Your presentation logic is complex enough to warrant a dedicated Presenter system

Basic Setup​

1. Add Component to Interactable Actor​

Blueprint:

  1. Open your interactable actor (Door, Item, NPC, etc.)
  2. Add Component → Search "ERPBaseInteractionFeedback"
  3. Add ERPBaseInteractionFeedbackComponent

C++:

UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Interaction")
UERPBaseInteractionFeedbackComponent* VisualFeedback;

AMyInteractableActor::AMyInteractableActor()
{
    VisualFeedback = CreateDefaultSubobject<UERPBaseInteractionFeedbackComponent>(TEXT("VisualFeedback"));
}

2. Configure Optional Sub-Components​

Outline (Optional):

  • Add a component that handles outline/highlight (e.g., PostProcessComponent, custom highlight component)
  • Assign it to OutlineComponent in the VisualFeedback component
  • The component will be enabled/disabled automatically based on focus

World-Space Widget (Optional):

  • Add a UWidgetComponent to your actor
  • Create a widget for the interaction prompt
  • Assign the WidgetComponent to WorldSpaceWidgetComponent
  • The widget will be shown/hidden automatically based on focus

3. Wire Events in PlayerController​

Bind to ERPInteractionComponent events:

Event BeginPlay:
  → Get Component: ERPInteractionComponent
  → Bind to OnCandidateChanged: HandleInteractionCandidateChanged
  → Bind to OnDescriptorChanged: HandleInteractionDescriptorChanged

Function: HandleInteractionCandidateChanged (Previous, New, ChannelId)
  → Branch: Is Valid (Previous)
    True:
      → Get Component by Class (Previous): ERPBaseInteractionFeedbackComponent
      → Branch: Is Valid (Component)
        True:
          → Call: SetInteractionFocused(false)
  
  → Branch: Is Valid (New)
    True:
      → Get Component by Class (New): ERPBaseInteractionFeedbackComponent
      → Branch: Is Valid (Component)
        True:
          → Call: SetInteractionFocused(true)

Function: HandleInteractionDescriptorChanged (Candidate, Descriptor, ChannelId)
  → Branch: Is Valid (Candidate)
    True:
      → Get Component by Class (Candidate): ERPBaseInteractionFeedbackComponent
      → Branch: Is Valid (Component)
        True:
          → Call: UpdateInteractionDescriptor(Descriptor)

Important Notes:

  • Always check if component is valid before calling functions
  • The component may not exist on all actors (it's optional)
  • Use GetComponentByClass to find the component dynamically
  • This pattern works with any subclass of ERPBaseInteractionFeedbackComponent

Integration with InteractionComponent​

Full Flow​

[ ERPPerceptionComponent ]
         ?
[ ERPInteractionComponent ]
         ? OnCandidateChanged / OnDescriptorChanged
[ PlayerController Event Bindings ]
         ? Calls SetInteractionFocused / UpdateInteractionDescriptor
[ ERPBaseInteractionFeedbackComponent ]
         ? Applies visual feedback
[ Outline + Widget ]

Why This Design?​

The VisualFeedback component is passive and actor-local:

  • It doesn't know about perception or interaction logic
  • It only reacts to explicit function calls
  • Multiple actors can have their own feedback components
  • The PlayerController orchestrates focus state changes

This keeps responsibilities separated and makes the system easier to debug.

Custom Visual Feedback​

Override Blueprint Events​

You can override these events in a Blueprint subclass:

OnInteractionFocused(bool bFocused)

  • Called when focus state changes
  • Add custom behavior (play FX, audio, animations)

Example:

Event: OnInteractionFocused (bFocused)
  ? Branch: bFocused
    True:
      ? Play Sound: FocusSound
      ? Spawn Emitter: FocusParticles
    False:
      ? Stop Sound
      ? Destroy Particles
  
  ? Parent: OnInteractionFocused (bFocused)

OnDescriptorUpdated(const FERPInteractionDescriptor& Descriptor)

  • Called when descriptor changes
  • Push data to your custom widgets

Example:

Event: OnDescriptorUpdated (Descriptor)
  ? Get Component: WidgetComponent
  ? Get User Widget Object
  ? Cast to: WBP_InteractionPrompt
  ? Set Prompt Text = Descriptor.PromptText
  ? Set Display Name = Descriptor.DisplayName
  
  ? Parent: OnDescriptorUpdated (Descriptor)

Override Native Implementation (C++)​

You can subclass the component and override:

class AMyCustomFeedbackComponent : public UERPBaseInteractionFeedbackComponent
{
    virtual void HandleFocusChanged(bool bFocused) override
    {
        Super::HandleFocusChanged(bFocused);
        
        // Your custom logic here
        if (bFocused)
        {
            PlayFocusVFX();
        }
        else
        {
            StopFocusVFX();
        }
    }
    
    virtual void HandleDescriptorUpdated(const FERPInteractionDescriptor& Descriptor) override
    {
        Super::HandleDescriptorUpdated(Descriptor);
        
        // Your custom logic here
        UpdateCustomWidget(Descriptor);
    }
};

Blueprint Examples​

Example 1: Simple Door with Outline​

Door Actor Setup:

  1. Add ERPBaseInteractionFeedbackComponent
  2. Add custom highlight component (e.g., outline material)
  3. Assign highlight component to OutlineComponent

Result:

  • Door automatically shows outline when player looks at it
  • No additional code needed

Example 2: Item with World-Space Prompt​

Item Actor Setup:

  1. Add ERPBaseInteractionFeedbackComponent
  2. Add WidgetComponent
  3. Create widget: WBP_ItemPrompt with Text Block
  4. Assign WidgetComponent to WorldSpaceWidgetComponent

Update widget content:

Event: OnDescriptorUpdated (Descriptor)
  ? Get Component: WorldSpaceWidgetComponent
  ? Get User Widget Object
  ? Cast to: WBP_ItemPrompt
  ? Set Text = Descriptor.PromptText

Result:

  • Widget shows when player looks at item
  • Text updates automatically based on descriptor

Example 3: NPC with Custom Animations​

NPC Actor Setup:

  1. Add ERPBaseInteractionFeedbackComponent
  2. Override OnInteractionFocused event
Event: OnInteractionFocused (bFocused)
  ? Branch: bFocused
    True:
      ? Play Animation: LookAtPlayer
      ? Enable Name Widget
    False:
      ? Play Animation: IdleAnimation
      ? Disable Name Widget

Result:

  • NPC plays special animation when player can interact
  • Name widget appears above NPC

C++ Examples​

Example 1: Custom Outline Effect​

UCLASS()
class AMyInteractableActor : public AActor, public IERPInteractable
{
    GENERATED_BODY()
    
public:
    AMyInteractableActor()
    {
        VisualFeedback = CreateDefaultSubobject<UERPBaseInteractionFeedbackComponent>(TEXT("VisualFeedback"));
        OutlineComponent = CreateDefaultSubobject<UPostProcessComponent>(TEXT("Outline"));
        
        // Configure outline material
        OutlineComponent->bEnabled = false;
        // ... setup outline material
    }
    
protected:
    virtual void BeginPlay() override
    {
        Super::BeginPlay();
        
        // Assign outline to feedback component
        VisualFeedback->OutlineComponent = OutlineComponent;
    }
    
private:
    UPROPERTY()
    UERPBaseInteractionFeedbackComponent* VisualFeedback;
    
    UPROPERTY()
    UPostProcessComponent* OutlineComponent;
};

Example 2: PlayerController Integration​

UCLASS()
class AMyPlayerController : public APlayerController
{
    GENERATED_BODY()
    
protected:
    virtual void BeginPlay() override
    {
        Super::BeginPlay();
        
        if (UERPInteractionComponent* InteractionComp = FindComponentByClass<UERPInteractionComponent>())
        {
            InteractionComp->OnCandidateChanged.AddDynamic(this, &AMyPlayerController::HandleCandidateChanged);
            InteractionComp->OnDescriptorChanged.AddDynamic(this, &AMyPlayerController::HandleDescriptorChanged);
        }
    }
    
private:
    UFUNCTION()
    void HandleCandidateChanged(AActor* Previous, AActor* New, FName ChannelId)
    {
        // Clear previous
        if (Previous)
        {
            if (UERPBaseInteractionFeedbackComponent* Feedback = Previous->FindComponentByClass<UERPBaseInteractionFeedbackComponent>())
            {
                Feedback->SetInteractionFocused(false);
            }
        }
        
        // Set new
        if (New)
        {
            if (UERPBaseInteractionFeedbackComponent* Feedback = New->FindComponentByClass<UERPBaseInteractionFeedbackComponent>())
            {
                Feedback->SetInteractionFocused(true);
            }
        }
    }
    
    UFUNCTION()
    void HandleDescriptorChanged(AActor* Candidate, const FERPInteractionDescriptor& Descriptor, FName ChannelId)
    {
        if (Candidate)
        {
            if (UERPBaseInteractionFeedbackComponent* Feedback = Candidate->FindComponentByClass<UERPBaseInteractionFeedbackComponent>())
            {
                Feedback->UpdateInteractionDescriptor(Descriptor);
            }
        }
    }
};

Common Patterns​

Pattern 1: Fallback to Screen-Space UI​

If an actor doesn't have ERPBaseInteractionFeedbackComponent, show screen-space UI instead:

Function: HandleCandidateChanged (Previous, New, ChannelId)
  ? Get Component: ERPBaseInteractionFeedbackComponent (New)
  ? Is Valid?
    True:
      ? Call: SetInteractionFocused(true)
      ? Hide Screen UI
    False:
      ? Show Screen UI (fallback)

Pattern 2: Conditional Feedback​

Enable/disable feedback based on game state:

Function: HandleCandidateChanged (Previous, New, ChannelId)
  ? Branch: Is In Menu
    False:
      ? Get Component: ERPBaseInteractionFeedbackComponent (New)
      ? Call: SetInteractionFocused(true)

Pattern 3: Multiple Feedback Types​

Use different feedback for different interaction types:

Event: OnInteractionFocused (bFocused)
  ? Get Interaction Tags (Owner)
  ? Has Tag: "Interact.Dangerous"
    True:
      ? Set Outline Color: Red
    False:
      ? Set Outline Color: White

Best Practices​

? DO​

  • Use ERPBaseInteractionFeedbackComponent for quick prototyping
  • Override Blueprint events for custom behavior
  • Safely check if component exists before calling
  • Keep visual feedback logic in the feedback component
  • Use this component for simple, actor-local visual effects

? DON'T​

  • Don't put interaction logic in the feedback component
  • Don't make the component subscribe to events automatically
  • Don't use this for complex, multi-actor presentation logic (use a Presenter instead)
  • Don't assume all interactables have this component
  • Don't create/destroy widgets dynamically in this component

Troubleshooting​

Visual feedback not showing​

Check:

  1. Is ERPBaseInteractionFeedbackComponent added to the interactable actor?
  2. Is SetInteractionFocused being called from PlayerController?
  3. Are sub-components (OutlineComponent, WorldSpaceWidgetComponent) assigned?
  4. Are sub-components properly configured?

Descriptor not updating widget​

Check:

  1. Is UpdateInteractionDescriptor being called?
  2. Is OnDescriptorUpdated event implemented?
  3. Is the widget reference valid?
  4. Is the widget structure matching your assumptions?

Outline/Widget not hiding​

Check:

  1. Is SetInteractionFocused(false) being called on the previous candidate?
  2. Is the previous candidate reference still valid?
  3. Are you handling OnCandidateChanged correctly?

Summary​

UERPBaseInteractionFeedbackComponent provides a clean, optional way to add visual feedback to interactable actors without writing boilerplate code.

Key Takeaways:

  • It's optional - only use if it fits your needs
  • It's passive - only reacts to explicit function calls
  • It's extensible - override events for custom behavior
  • It's simple - designed for quick setup and prototyping

For more complex presentation needs, consider implementing a custom Presenter system instead.
