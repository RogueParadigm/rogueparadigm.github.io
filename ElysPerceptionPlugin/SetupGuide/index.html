Pipeline Stages Explainedâ€‹

1. Samplerâ€‹

What it does: Gathers potential candidates from the world

Example: ERPSphereOverlapSampler does a sphere overlap to find nearby actors

Analogy: Casting a fishing net

2. Filtersâ€‹

What it does: Removes invalid candidates (ALL filters must pass)

Example: ERPInteractableFilter checks if actor implements IERPInteractable interface

Analogy: Throwing back fish that are too small

3. Scorersâ€‹

What it does: Assigns a score to each remaining candidate

Example: ERPDistanceScorer scores by distance (closer = lower score = better)

Analogy: Rating fish by size

4. Aggregatorâ€‹

What it does: Combines multiple scorer outputs into one final score

Example: ERPIdentityAggregator just passes through (for single scorer)

Analogy: Weighing all fish characteristics

5. Resolverâ€‹

What it does: Picks the winning candidate based on final scores

Example: ERPLowestScoreResolver picks the candidate with lowest score

Analogy: Selecting the best fish

Multi-Channel Systemâ€‹

The plugin supports multiple independent channels:

  • Interaction Channel â€” For detecting interactable objects (doors, items, NPCs)
  • Target Channel â€” For combat targeting
  • Custom Channels â€” Add your own (healing targets, etc.)

Each channel:

  • Runs its own pipeline
  • Maintains its own best candidate
  • Fires its own events

Example: You can have an enemy targeted (Target channel) while also detecting a nearby door (Interaction channel) at the same time.

      â†’ Get Component by Class (Candidate): ERPBaseInteractionFeedbackComponent
      â†’ Branch: Is Valid (Component)

What this does:

  • When a new interactable is detected, SetInteractionFocused(true) is called on its feedback component
  • When focus is lost, SetInteractionFocused(false) is called on the previous candidate
  • When the descriptor changes (e.g., door state changes from "Open" to "Close"), the feedback component is updated

No UI logic in PlayerController! All visual behavior is handled by the feedback component on the interactable actor.

C++ Implementationâ€‹

// PlayerController.h
#include "Core/ERPPerceptionComponent.h"
#include "Interaction/ERPInteractionComponent.h"

UCLASS()
class AMyPlayerController : public APlayerController
{
    GENERATED_BODY()

public:
    AMyPlayerController();

protected:
    virtual void BeginPlay() override;

private:
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Perception", meta = (AllowPrivateAccess = "true"))
    UERPPerceptionComponent* PerceptionComponent;

    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Interaction", meta = (AllowPrivateAccess = "true"))
    UERPInteractionComponent* InteractionComponent;

    UFUNCTION()
    void OnCandidateChanged(AActor* PreviousCandidate, AActor* NewCandidate, FName ChannelId);

    UFUNCTION()
    void OnDescriptorChanged(AActor* Candidate, const FERPInteractionDescriptor& Descriptor, FName ChannelId);
};

// PlayerController.cpp
#include "Interaction/ERPBaseInteractionFeedbackComponent.h"

AMyPlayerController::AMyPlayerController()
{
    PerceptionComponent = CreateDefaultSubobject<UERPPerceptionComponent>(TEXT("PerceptionComponent"));
    InteractionComponent = CreateDefaultSubobject<UERPInteractionComponent>(TEXT("InteractionComponent"));
    InteractionComponent->ChannelId = "Interaction";
}

void AMyPlayerController::BeginPlay()
{
    Super::BeginPlay();

    // Bind interaction events
    if (InteractionComponent)
    {
        InteractionComponent->OnCandidateChanged.AddDynamic(this, &AMyPlayerController::OnCandidateChanged);
        InteractionComponent->OnDescriptorChanged.AddDynamic(this, &AMyPlayerController::OnDescriptorChanged);
    }
}

void AMyPlayerController::OnCandidateChanged(AActor* PreviousCandidate, AActor* NewCandidate, FName ChannelId)
{
    // Unfocus previous candidate
    if (PreviousCandidate)
    {
        if (UERPBaseInteractionFeedbackComponent* FeedbackComp = PreviousCandidate->FindComponentByClass<UERPBaseInteractionFeedbackComponent>())
        {
            FeedbackComp->SetInteractionFocused(false);
        }
    }

    // Focus new candidate
    if (NewCandidate)
    {
        if (UERPBaseInteractionFeedbackComponent* FeedbackComp = NewCandidate->FindComponentByClass<UERPBaseInteractionFeedbackComponent>())
        {
            FeedbackComp->SetInteractionFocused(true);
        }
    }
}

void AMyPlayerController::OnDescriptorChanged(AActor* Candidate, const FERPInteractionDescriptor& Descriptor, FName ChannelId)
{
    if (Candidate)
    {
        if (UERPBaseInteractionFeedbackComponent* FeedbackComp = Candidate->FindComponentByClass<UERPBaseInteractionFeedbackComponent>())
        {
            FeedbackComp->UpdateInteractionDescriptor(Descriptor);
        }
    }
}

Key Points:

  • ERPInteractionComponent automatically subscribes to perception events for the specified channel
  • It validates candidates using CanBeInteractedWith and optional InteractionRules
  • It resolves descriptors by calling GetInteractionDescriptor on the candidate
  • Events fire only when state actually changes (candidate or descriptor)

UCLASS()
class AInteractableDoor : public AActor, public IERPInteractable

bool AInteractableDoor::CanBeInteractedWith_Implementation(AActor* InteractingActor)

    return !bIsLocked;

void AInteractableDoor::GetInteractionDescriptor_Implementation(FERPInteractionDescriptor& OutDescriptor)

Troubleshootingâ€‹

No prompt appears when approaching objectâ€‹

Check:

  • Does the object implement IERPInteractable interface?
  • Does CanBeInteractedWith return True?
  • Is the object within range? (Default: 300 units)
  • Is the Trace Channel correct? (Try ECC_Visibility or ECC_WorldDynamic)
  • Are events bound in BeginPlay?

Debug:

Event Tick
  â†’ Get Component: PerceptionComponent
  â†’ Get Channel Candidate ("Interaction")
  â†’ Print String: Result

Events not firingâ€‹

Check:

  • Is ERPChannelListenerComponent's Channel Id set to "Interaction"?
  • Are events bound in BeginPlay?
  • Is the PlayerController the local player? (not AI, not server-only)

Debug: Enable bDebugPerception on ERPPerceptionComponent (Details panel)

Object detected from too far/too closeâ€‹

Adjust:

  • DefaultSamplingRange on ERPPerceptionComponent (affects all channels)
  • Or: Use custom Sampler with specific radius

Multiple objects detected, wrong one selectedâ€‹

Check:

  • Resolver is ERPLowestScoreResolver (picks closest)
  • Scorer is ERPDistanceScorer with Normalize By Range = true

Improve:

  • Add angle-based scoring (see Advanced Customization)
  • Add custom filters (line-of-sight, etc.)
