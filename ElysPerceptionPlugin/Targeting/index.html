Overview‚Äã

What is the Targeting System?‚Äã

The targeting system allows your player to detect and lock onto enemies for combat:

  • Lock-on targeting (Dark Souls style) - Lock camera to target
  • Soft targeting (auto-aim assistance) - Subtle aim adjustment
  • Priority targeting (threat-based) - Auto-select most dangerous enemy
  • Multi-target management - Track multiple threats

How It Works‚Äã

The targeting system uses the same pipeline as interaction, but with different configuration:

[Perception Component] ‚Üí Sampler ‚Üí Filter ‚Üí Scorer ‚Üí Aggregator ‚Üí Resolver ‚Üí [Locked Target]
     (enemies)           (sphere)  (hostile)  (threat) (weighted)  (highest)

Key differences from interaction:

AspectInteractionTargeting
Range200-500 units2000-5000 units
ScoringDistance onlyDistance + Angle + Threat
Lock behaviorNo lockPersistent lock
CameraNo camera changeCamera follows target

Basic Lock-On System‚Äã

// Input Action: ToggleTargetLock (Gamepad: R3 / Keyboard: Tab)

Event ToggleTargetLock:
  ‚Üí Branch: IsTargetLocked
  
  [True - Unlock]:
    ‚Üí Set IsTargetLocked = false
    ‚Üí Set LockedTarget = None
    ‚Üí (Optional) Reset Camera
    ‚Üí Event "OnTargetUnlocked"
    
  [False - Lock]:
    ‚Üí Get Perception Component ‚Üí Get Channel Candidate ("Targeting")
    ‚Üí Branch: Is Valid?
      [True]:
        ‚Üí Set LockedTarget = Candidate
        ‚Üí Set IsTargetLocked = true
        ‚Üí Event "OnTargetLocked" (pass LockedTarget)
      [False]:
        ‚Üí (Optional) Play "No Target" feedback

Maintaining the Lock‚Äã

// In Tick or Timer (every 0.1s)

Branch: IsTargetLocked AND LockedTarget is Valid
  [True]:
    ‚Üí Get Distance to LockedTarget
    ‚Üí Branch: Distance > MaxLockDistance (e.g., 4000)
      [True]: ‚Üí Call UnlockTarget
      
    ‚Üí (Optional) Line Trace to LockedTarget
    ‚Üí Branch: Blocked for > 2 seconds?
      [True]: ‚Üí Call UnlockTarget
      
  [False]:
    ‚Üí If IsTargetLocked but LockedTarget invalid:
      ‚Üí Call UnlockTarget

C++ Implementation‚Äã

// TargetingComponent.h
UCLASS(ClassGroup=(Custom), meta=(BlueprintSpawnableComponent))
class UERPTargetingComponent : public UActorComponent
{
    GENERATED_BODY()

public:
    UPROPERTY(BlueprintReadOnly, Category = "Targeting")
    bool bIsTargetLocked = false;
    
    UPROPERTY(BlueprintReadOnly, Category = "Targeting")
    TWeakObjectPtr<AActor> LockedTarget;
    
    UPROPERTY(EditDefaultsOnly, Category = "Targeting")
    float MaxLockDistance = 4000.0f;
    
    UPROPERTY(EditDefaultsOnly, Category = "Targeting")
    float LockBreakTime = 2.0f;
    
    UFUNCTION(BlueprintCallable, Category = "Targeting")
    bool TryLockTarget();
    
    UFUNCTION(BlueprintCallable, Category = "Targeting")
    void UnlockTarget();
    
    DECLARE_DYNAMIC_MULTICAST_DELEGATE_OneParam(FOnTargetLocked, AActor*, Target);
    UPROPERTY(BlueprintAssignable)
    FOnTargetLocked OnTargetLocked;
    
    DECLARE_DYNAMIC_MULTICAST_DELEGATE(FOnTargetUnlocked);
    UPROPERTY(BlueprintAssignable)
    FOnTargetUnlocked OnTargetUnlocked;

protected:
    UPROPERTY()
    UERPPerceptionComponent* PerceptionComponent;
    
    FName TargetingChannelId = "Targeting";
    float TimeTargetObstructed = 0.0f;
};

// TargetingComponent.cpp
bool UERPTargetingComponent::TryLockTarget()
{
    if (!PerceptionComponent) return false;
    
    if (AActor* Candidate = PerceptionComponent->GetChannelCandidate(TargetingChannelId))
    {
        LockedTarget = Candidate;
        bIsTargetLocked = true;
        TimeTargetObstructed = 0.0f;
        OnTargetLocked.Broadcast(Candidate);
        return true;
    }
    return false;
}

void UERPTargetingComponent::UnlockTarget()
{
    AActor* PreviousTarget = LockedTarget.Get();
    LockedTarget.Reset();
    bIsTargetLocked = false;
    TimeTargetObstructed = 0.0f;
    OnTargetUnlocked.Broadcast();
}

Target Switching‚Äã

Directional Switching‚Äã

Switch to the next target in a direction (left/right based on stick input):

// Input: Right Stick X-Axis

Event SwitchTargetDirection (Axis Value):
  ‚Üí Branch: IsTargetLocked AND Abs(AxisValue) > 0.5
  
  [True]:
    ‚Üí Get All Actors in Targeting Channel (from Perception)
    ‚Üí Filter: Remove current LockedTarget
    ‚Üí Filter: Remove actors not in direction
    
    // Get direction from camera
    ‚Üí Get Camera Right Vector
    ‚Üí For Each Actor:
      ‚Üí Dot Product: (ActorLocation - PlayerLocation).Normalized ‚Ä¢ CameraRight
      ‚Üí If AxisValue > 0: Keep actors where Dot > 0.3
      ‚Üí If AxisValue < 0: Keep actors where Dot < -0.3
    
    ‚Üí Sort remaining by angle from center
    ‚Üí Set LockedTarget = First actor (closest to center in that direction)

C++ Target Switching‚Äã

void UERPTargetingComponent::SwitchTarget(float Direction)
{
    if (!bIsTargetLocked || !PerceptionComponent) return;
    
    AActor* Owner = GetOwner();
    if (!Owner) return;
    
    // Get camera right vector for directional reference
    APlayerController* PC = Cast<APlayerController>(Owner->GetInstigatorController());
    if (!PC) return;
    
    FVector CameraRight = PC->PlayerCameraManager->GetCameraRotation().RotateVector(FVector::RightVector);
    FVector PlayerLocation = Owner->GetActorLocation();
    
    AActor* BestCandidate = nullptr;
    float BestScore = TNumericLimits<float>::Max();
    
    // Get all current candidates from perception
    TArray<FERPPipelineCandidate> Candidates;
    // Note: You'd need to expose GetAllCandidates or iterate registered actors
    
    for (const auto& Candidate : Candidates)
    {
        if (Candidate.Actor == LockedTarget.Get()) continue;
        
        FVector ToTarget = (Candidate.Actor->GetActorLocation() - PlayerLocation).GetSafeNormal();
        float DotRight = FVector::DotProduct(ToTarget, CameraRight);
        
        // Check if target is in the requested direction
        if ((Direction > 0 && DotRight > 0.2f) || (Direction < 0 && DotRight < -0.2f))
        {
            float Score = FMath::Abs(DotRight); // Prefer targets more to the side
            if (Score < BestScore)
            {
                BestScore = Score;
                BestCandidate = Candidate.Actor;
            }
        }
    }
    
    if (BestCandidate)
    {
        LockedTarget = BestCandidate;
        OnTargetLocked.Broadcast(BestCandidate);
    }
}

Multi-Factor Scoring‚Äã

Threat-Based Scoring‚Äã

Create a custom scorer that factors in threat level:

// ERPThreatScorer.h
UCLASS(Blueprintable, EditInlineNew, DefaultToInstanced)
class UERPThreatScorer : public UERPBaseScorer
{
    GENERATED_BODY()

public:
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Scoring")
    float ThreatWeight = 1.0f;
    
protected:
    virtual float Score_Implementation(const FERPPipelineContext& Context, AActor* Candidate) override
    {
        // Check for threat interface
        if (IERPThreatSource* ThreatSource = Cast<IERPThreatSource>(Candidate))
        {
            float ThreatLevel = ThreatSource->GetThreatLevel(); // 0.0 - 1.0
            // Invert so higher threat = lower score (better)
            return (1.0f - ThreatLevel) * ThreatWeight;
        }
        return 0.5f * ThreatWeight; // Neutral threat if no interface
    }
};

Weighted Aggregator Setup‚Äã

Combine multiple scores with weights:

// Pipeline Configuration

Scorers:
  1. Distance Scorer (Weight: 0.4)
     - Closer = better
  
  2. Cone Scorer (Weight: 0.3)  
     - More centered in view = better
  
  3. Threat Scorer (Weight: 0.3)
     - Higher threat level = better (prioritize dangerous enemies)

Aggregator: Weighted Aggregator
  - Automatically uses scorer weights
  - Final Score = Œ£(ScoreN √ó WeightN) / Œ£(WeightN)

Example: Boss Priority‚Äã

Make bosses always highest priority:

float UERPThreatScorer::Score_Implementation(const FERPPipelineContext& Context, AActor* Candidate)
{
    // Bosses always get priority (lowest score)
    if (Candidate->ActorHasTag("Boss"))
    {
        return -1000.0f; // Guaranteed to win
    }
    
    // Elite enemies get high priority
    if (Candidate->ActorHasTag("Elite"))
    {
        return 0.1f;
    }
    
    // Regular enemies based on interface
    if (IERPThreatSource* ThreatSource = Cast<IERPThreatSource>(Candidate))
    {
        return 1.0f - ThreatSource->GetThreatLevel();
    }
    
    return 0.5f;
}

Best Practices‚Äã

Performance‚Äã

  • Use appropriate update rates (0.1s for targeting checks, not every frame)
  • Cache perception results, don't query every frame
  • Limit sampler range to what's actually needed

UX‚Äã

  • Provide clear visual feedback for lock state
  • Allow easy unlock (same button, or movement-based)
  • Consider "sticky" targeting that doesn't break easily
  • Audio feedback for lock/unlock

Multiplayer‚Äã

  • Target locking is client-side only
  • Don't replicate lock state
  • Server validates attacks independently
