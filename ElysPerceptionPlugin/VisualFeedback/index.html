Visual Feedback

Optional visual components that enhance the perception system with visual feedback.

Overview​

Visual components are optional enhancements that work alongside perception. They contain no gameplay logic and are fully decoupled from the core system.

Design Philosophy​

  • Purely Visual: No gameplay logic or state management
  • Fully Decoupled: Don't depend on perception or targeting systems
  • Reusable: Work with any actor, in any context
  • Optional: Can be added or removed without affecting gameplay

Outline Component​

The ERPOutlineComponent adds mesh overlay outlines to actors at runtime.

Use Cases​

  • Interaction highlighting (show outline when object is interactable)
  • Target indicators (highlight current target in combat)
  • Focus states (different colors for primary/secondary focus)
  • Quest markers (highlight quest-related objects)
  • Accessibility (help players identify interactive elements)

How It Works​

Uses Unreal's Overlay Material system:

  1. Finds a mesh component on the actor
  2. Creates Dynamic Material Instance from configured Overlay Material
  3. Applies/removes overlay material at runtime
  4. Updates material parameters (color, thickness) dynamically

Setup​

1. Add Component​

Blueprint:

  • Open your actor Blueprint (door, item, enemy)
  • Add Component → Search for ERPOutlineComponent

C++:

OutlineComponent = CreateDefaultSubobject<UERPOutlineComponent>(TEXT("Outline"));

2. Configure​

In component details:

  • Overlay Material: Select your outline material (Unlit + Opaque)
  • Default Color: Initial outline color (default: cyan)
  • Default Thickness: Initial outline width (default: 1.0)

3. Create Overlay Material​

Create a material for mesh overlays:

Material Setup:

  • Shading Model: Unlit
  • Blend Mode: Opaque
  • Parameters:
    • OutlineColor (Vector Parameter)
    • OutlineThickness (Scalar Parameter)

Basic Material Graph:

OutlineColor (Parameter)
  → Emissive Color

OutlineThickness (Parameter)
  → Multiply with Normal
  → Add to World Position Offset

A sample outline material is included in plugin content.

API Reference​

EnableOutline()​

Activates the outline effect.

Blueprint:

Get Component: OutlineComponent
  → Enable Outline

C++:

OutlineComponent->EnableOutline();

DisableOutline()​

Deactivates the outline effect.

Blueprint:

Get Component: OutlineComponent
  → Disable Outline

C++:

OutlineComponent->DisableOutline();

SetOutlineColor(FLinearColor Color)​

Changes the outline color at runtime.

Blueprint:

Get Component: OutlineComponent
  → Set Outline Color
    → Color: (R=1, G=0, B=0, A=1) // Red

C++:

OutlineComponent->SetOutlineColor(FLinearColor::Red);

SetOutlineThickness(float Thickness)​

Changes the outline width at runtime.

Blueprint:

Get Component: OutlineComponent
  → Set Outline Thickness
    → Thickness: 2.0

C++:

OutlineComponent->SetOutlineThickness(2.0f);

Integration Examples​

Target Highlighting​

Blueprint:

Event: OnCandidateAcquired (Target)
  → Get Component by Class (Target, ERPOutlineComponent)
  → Enable Outline
  → Set Outline Color: Orange

Event: OnCandidateLost (Target)
  → Get Component by Class (Target, ERPOutlineComponent)
  → Disable Outline

Interaction Highlighting​

Blueprint:

Event: OnDescriptorChanged (Interactable, Descriptor)
  → Get Component by Class (Interactable, ERPOutlineComponent)
  → Enable Outline
  → Set Outline Color: Cyan

Event: Candidate Lost
  → Get Component by Class (Interactable, ERPOutlineComponent)
  → Disable Outline

Focus-Based Colors​

Blueprint:

Event: OnCandidateAcquired (Candidate)
  → Get Component by Class (Candidate, ERPOutlineComponent)
  → Branch: Has Focus?
    True: → Set Outline Color: Orange (primary)
    False: → Set Outline Color: Blue (secondary)

Best Practices​

Performance​

  • Outlines use overlay materials which have minimal performance impact
  • Disable outline when not needed to avoid unnecessary material updates
  • Use simple outline materials (avoid complex shaders)

Visual Design​

  • Use distinct colors for different interaction types
  • Keep outline thickness reasonable (1.0-3.0 range)
  • Test visibility on various backgrounds
  • Consider colorblind-friendly palettes

Integration​

  • Add outline components to interactable actors
  • Control via perception events (acquired/lost)
  • Use different colors for different focus states
  • Disable when interaction is no longer available

Advanced Usage​

Animated Outlines​

Use material parameters with Timeline:

Blueprint:

Timeline: PulseOutline
  → Track: Alpha (0.0 to 1.0, looping)
  → Update:
    → Get Outline Component
    → Lerp Color: Cyan to White (Alpha)
    → Set Outline Color

Conditional Outlines​

Only show outline if player has certain item:

Blueprint:

Function: Can Be Interacted With (Player)
  → Branch: Player Has Key?
    True:
      → Enable Outline
      → Return True
    False:
      → Disable Outline
      → Return False
