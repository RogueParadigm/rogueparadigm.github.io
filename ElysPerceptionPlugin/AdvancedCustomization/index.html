Advanced Customization Guide

This guide covers creating custom pipeline components and extending the ElysPerceptionPlugin for advanced use cases.

Overview‚Äã

When to Customize‚Äã

Create custom components when:

  • Default components don't fit your needs
  • You need game-specific logic (health priority, threat levels)
  • You want special filtering (line of sight, visibility)
  • You need unique scoring (custom gameplay metrics)

Extension Points‚Äã

The plugin provides base classes for each pipeline stage:

  • UERPSamplerBase - Custom candidate gathering
  • UERPFilterBase - Custom filtering logic
  • UERPScorerBase - Custom scoring logic
  • UERPAggregatorBase - Custom score combination
  • UERPResolverBase - Custom winner selection

Implementation Options‚Äã

BlueprintC++
Quick prototypingBest performance
Designer-friendlyType safety
No compilationFull engine access
Limited debuggingAdvanced debugging

Choose based on:

  • Blueprint: Simple logic, rapid iteration, designer-owned
  • C++: Complex logic, performance-critical, programmer-owned

Creating Custom Samplers‚Äã

Purpose‚Äã

Samplers gather potential candidates from the world. Customize when you need:

  • Different spatial queries (raycast, box overlap, etc.)
  • Non-spatial queries (GameState actor lists, etc.)
  • Cached/optimized candidate lists

Base Class‚Äã

UCLASS(Abstract, BlueprintType, Blueprintable, EditInlineNew, DefaultToInstanced)
class ELYSPERCEPTIONPLUGIN_API UERPSamplerBase : public UObject
{
    GENERATED_BODY()

public:
    UFUNCTION(BlueprintNativeEvent, Category = "ElysPerception|Pipeline")
    void Sample(const FERPPerceptionContext& Context, float Range, TArray<AActor*>& OutCandidates) const;
};

C++ Example: Line Trace Sampler‚Äã

ERPLineTraceSampler.h:

#pragma once

#include "CoreMinimal.h"
#include "Pipeline/Samplers/ERPSamplerBase.h"
#include "ERPLineTraceSampler.generated.h"

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPLineTraceSampler : public UERPSamplerBase
{
    GENERATED_BODY()

public:
    virtual void Sample_Implementation(
        const FERPPerceptionContext& Context,
        float Range,
        TArray<AActor*>& OutCandidates) const override;

    UPROPERTY(EditAnywhere, Category = "Sampling")
    TEnumAsByte<ECollisionChannel> TraceChannel = ECC_Visibility;
    
    UPROPERTY(EditAnywhere, Category = "Sampling")
    bool bTraceComplex = false;
};

ERPLineTraceSampler.cpp:

#include "ERPLineTraceSampler.h"
#include "Engine/World.h"

void UERPLineTraceSampler::Sample_Implementation(
    const FERPPerceptionContext& Context,
    float Range,
    TArray<AActor*>& OutCandidates) const
{
    OutCandidates.Empty();
    
    if (!Context.ContextActor)
        return;
    
    UWorld* World = Context.ContextActor->GetWorld();
    if (!World)
        return;
    
    // Line trace in forward direction
    FVector Start = Context.Origin;
    FVector End = Context.Origin + Context.Forward * Range;
    
    FCollisionQueryParams QueryParams;
    QueryParams.bTraceComplex = bTraceComplex;
    QueryParams.AddIgnoredActor(Context.ContextActor);
    
    FHitResult HitResult;
    if (World->LineTraceSingleByChannel(HitResult, Start, End, TraceChannel, QueryParams))
    {
        if (HitResult.GetActor())
        {
            OutCandidates.Add(HitResult.GetActor());
        }
    }
}

Create Blueprint class:

  1. Content Browser ‚Üí Right-click ‚Üí Blueprint Class

  3. Name: BP_NearbyActorsSampler

Implement Sample:

  ‚Üí Get All Actors of Class: YourActorClass
  ‚Üí For Each Actor:
    ‚Üí Get Distance (Context.Origin to Actor)
    ‚Üí Branch: Distance < Range
      True: ‚Üí Add to OutCandidates

C++ Example: Line of Sight Filter‚Äã

ERPLineOfSightFilter.h:

#pragma once

#include "CoreMinimal.h"
#include "Pipeline/Filters/ERPFilterBase.h"
#include "ERPLineOfSightFilter.generated.h"

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPLineOfSightFilter : public UERPFilterBase
{
    GENERATED_BODY()

public:
    virtual bool Passes_Implementation(
        const FERPPerceptionContext& Context,
        AActor* Candidate) const override;

    UPROPERTY(EditAnywhere, Category = "Filter")
    TEnumAsByte<ECollisionChannel> TraceChannel = ECC_Visibility;
    
    UPROPERTY(EditAnywhere, Category = "Filter")
    bool bRequireFullyVisible = true;
};

ERPLineOfSightFilter.cpp:

#include "ERPLineOfSightFilter.h"
#include "Engine/World.h"

bool UERPLineOfSightFilter::Passes_Implementation(
    const FERPPerceptionContext& Context,
    AActor* Candidate) const
{
    if (!Context.ContextActor || !Candidate)
        return false;
    
    UWorld* World = Context.ContextActor->GetWorld();
    if (!World)
        return false;
    
    FVector Start = Context.Origin;
    FVector End = Candidate->GetActorLocation();
    
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(Context.ContextActor);
    QueryParams.AddIgnoredActor(Candidate);
    
    FHitResult HitResult;
    bool bHit = World->LineTraceSingleByChannel(
        HitResult,
        Start,
        End,
        TraceChannel,
        QueryParams
    );
    
    // Pass if no hit (clear line of sight)
    return !bHit;
}

C++ Example: Health Priority Scorer‚Äã

ERPHealthScorer.h:

#pragma once

#include "CoreMinimal.h"
#include "Pipeline/Scorers/ERPScorerBase.h"
#include "ERPHealthScorer.generated.h"

// Custom interface for actors with health
UINTERFACE(MinimalAPI, Blueprintable)
class UHealthComponent : public UInterface
{
    GENERATED_BODY()
};

class IHealthComponent
{
    GENERATED_BODY()

public:
    UFUNCTION(BlueprintNativeEvent, Category = "Health")
    float GetHealthPercent() const;
};

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPHealthScorer : public UERPScorerBase
{
    GENERATED_BODY()

public:
    virtual float Score_Implementation(
        const FERPPerceptionContext& Context,
        AActor* Candidate) const override;

    // Lower health = lower score (better)
    UPROPERTY(EditAnywhere, Category = "Scoring")
    bool bPrioritizeLowHealth = true;
};

ERPHealthScorer.cpp:

#include "ERPHealthScorer.h"

float UERPHealthScorer::Score_Implementation(
    const FERPPerceptionContext& Context,
    AActor* Candidate) const
{
    if (!Candidate || !Candidate->Implements<UHealthComponent>())
        return 1.0f;  // Neutral score
    
    float HealthPercent = IHealthComponent::Execute_GetHealthPercent(Candidate);
    
    if (bPrioritizeLowHealth)
    {
        // Lower health = lower score (better)
        return HealthPercent;  // 0.0 (low health) to 1.0 (full health)
    }
    else
    {
        // Higher health = lower score (better)
        return 1.0f - HealthPercent;
    }
}

Blueprint Example: Threat Level Scorer‚Äã

// BP_ThreatScorer (inherits from ERPScorerBase)

Function: Score (Context, Candidate)
  ‚Üí Cast Candidate to Enemy
  ‚Üí Branch: Cast Succeeded
    True:
      ‚Üí Get Threat Level (Enemy)  // Returns 1-10
      ‚Üí Divide by 10  // Normalize to 0.0-1.0
      ‚Üí Return (1.0 - normalized)  // Invert: high threat = low score
    False:
      ‚Üí Return: 1.0  // Neutral score

C++ Example: Weighted Aggregator‚Äã

ERPWeightedAggregator.h:

#pragma once

#include "CoreMinimal.h"
#include "Pipeline/Aggregators/ERPAggregatorBase.h"
#include "ERPWeightedAggregator.generated.h"

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPWeightedAggregator : public UERPAggregatorBase
{
    GENERATED_BODY()

public:
    virtual float Aggregate_Implementation(const TArray<float>& Scores) const override;

    // Weights for each scorer (must match scorer count)
    UPROPERTY(EditAnywhere, Category = "Aggregation")
    TArray<float> Weights;
};

ERPWeightedAggregator.cpp:

#include "ERPWeightedAggregator.h"

float UERPWeightedAggregator::Aggregate_Implementation(const TArray<float>& Scores) const
{
    if (Scores.Num() == 0)
        return 0.0f;
    
    float WeightedSum = 0.0f;
    float TotalWeight = 0.0f;
    
    for (int32 i = 0; i < Scores.Num(); ++i)
    {
        float Weight = (i < Weights.Num()) ? Weights[i] : 1.0f;
        WeightedSum += Scores[i] * Weight;
        TotalWeight += Weight;
    }
    
    return (TotalWeight > 0.0f) ? (WeightedSum / TotalWeight) : 0.0f;
}

Creating Custom Resolvers‚Äã

Purpose‚Äã

Resolvers select the winning candidate. Customize when you need:

  • Closest to target value
  • Threshold-based selection
  • Hysteresis (sticky targets)

    GENERATED_BODY()

        const TArray<FERPPipelineScoredCandidate>& ScoredCandidates,
        FERPPipelineScoredCandidate& OutBest) const;
};

C++ Example: Threshold Resolver‚Äã

ERPThresholdResolver.h:

#pragma once

#include "CoreMinimal.h"
#include "Pipeline/Resolvers/ERPResolverBase.h"
#include "Pipeline/ERPPerceptionPipelineTypes.h"
#include "ERPThresholdResolver.generated.h"

UCLASS(Blueprintable, EditInlineNew)
class YOURGAME_API UERPThresholdResolver : public UERPResolverBase
{
    GENERATED_BODY()

public:
    virtual bool Resolve_Implementation(
        const TArray<FERPPipelineScoredCandidate>& ScoredCandidates,
        FERPPipelineScoredCandidate& OutBest) const override;

    // Only select if score is below this threshold
    UPROPERTY(EditAnywhere, Category = "Resolution")
    float ScoreThreshold = 0.5f;
    
    // If true, select lowest; if false, select highest
    UPROPERTY(EditAnywhere, Category = "Resolution")
    bool bSelectLowest = true;
};

ERPThresholdResolver.cpp:

#include "ERPThresholdResolver.h"

bool UERPThresholdResolver::Resolve_Implementation(
    const TArray<FERPPipelineScoredCandidate>& ScoredCandidates,
    FERPPipelineScoredCandidate& OutBest) const
{
    if (ScoredCandidates.Num() == 0)
        return false;
    
    bool bFoundValid = false;
    float BestScore = bSelectLowest ? FLT_MAX : -FLT_MAX;
    
    for (const FERPPipelineScoredCandidate& Scored : ScoredCandidates)
    {
        // Check threshold
        if (bSelectLowest && Scored.Score >= ScoreThreshold)
            continue;
        if (!bSelectLowest && Scored.Score <= ScoreThreshold)
            continue;
        
        // Check if better
        bool bIsBetter = bSelectLowest
            ? (Scored.Score < BestScore)
            : (Scored.Score > BestScore);
        
        if (!bFoundValid || bIsBetter)
        {
            OutBest = Scored;
            BestScore = Scored.Score;
            bFoundValid = true;
        }
    }
    
    return bFoundValid;
}

Blueprint vs C++‚Äã

Blueprint Custom Components‚Äã

Pros:

  • ‚úÖ Fast prototyping
  • ‚úÖ No compilation needed
  • ‚úÖ Designer-friendly
  • ‚úÖ Visual debugging

Cons:

  • ‚ùå Slower performance
  • ‚ùå Limited debugging
  • ‚ùå No compile-time safety
  • ‚ùå Can't access all engine APIs

Best for:

  • Simple logic
  • Rapid iteration
  • Designer-owned components
  • Prototypes

C++ Custom Components‚Äã

Pros:

  • ‚úÖ Best performance
  • ‚úÖ Full engine access
  • ‚úÖ Type safety
  • ‚úÖ Advanced debugging
  • ‚úÖ Code reuse

Cons:

  • ‚ùå Requires compilation
  • ‚ùå Programmer-only
  • ‚ùå Slower iteration

Best for:

  • Performance-critical code
  • Complex logic
  • Production code
  • Reusable libraries

Best Practices‚Äã

Performance‚Äã

  • ‚úÖ Do: Normalize scores to 0.0-1.0 range
  • ‚úÖ Do: Early-reject in filters
  • ‚úÖ Do: Cache expensive calculations
  • ‚úÖ Do: Use const references
  • ‚ùå Don't: Perform expensive operations in scorers
  • ‚ùå Don't: Allocate memory in hot paths
  • ‚ùå Don't: Use complex BP loops

Architecture‚Äã

  • ‚úÖ Do: Keep components stateless
  • ‚úÖ Do: Use context data
  • ‚úÖ Do: Return neutral values on error
  • ‚úÖ Do: Validate inputs
  • ‚ùå Don't: Store state between ticks
  • ‚ùå Don't: Modify candidates
  • ‚ùå Don't: Access external systems

Debugging‚Äã

  • ‚úÖ Do: Log meaningful information
  • ‚úÖ Do: Add debug visualization
  • ‚úÖ Do: Test in isolation
  • ‚úÖ Do: Document assumptions
  • ‚ùå Don't: Leave debug spam
  • ‚ùå Don't: Skip edge case testing

Design‚Äã

  • ‚úÖ Do: Single responsibility per component
  • ‚úÖ Do: Make components reusable
  • ‚úÖ Do: Expose configuration options
  • ‚úÖ Do: Provide sensible defaults
  • ‚ùå Don't: Couple to specific actors
  • ‚ùå Don't: Hardcode values
  • ‚ùå Don't: Mix concerns
