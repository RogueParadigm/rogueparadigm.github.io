Visual Components & Helpers

This guide covers optional visual components that enhance the perception system with visual feedback and quality-of-life features.

ðŸ’¡ Note: These components are completely optional and separate from the core perception system. They provide visual enhancements but contain no gameplay logic.

Overviewâ€‹

Visual components are optional enhancement components that work alongside the perception system. They don't contain gameplay logic or decision-making - they're purely for visual feedback and developer convenience.

Design Philosophyâ€‹

Visual helper components follow these principles:

  • Purely Visual: No gameplay logic or state management
  • Fully Decoupled: Don't depend on perception or targeting systems
  • Reusable: Work with any actor, in any context
  • Optional: Can be added or removed without affecting gameplay

Outline Component (ERP_OutlineComponent)â€‹

The ERP_OutlineComponent (or UERPOutlineComponent in code) is a lightweight visual helper that adds mesh overlay outlines to actors at runtime.

Use Casesâ€‹

  • Interaction Highlighting: Show outline when object is interactable
  • Target Indicators: Highlight the current target in combat
  • Focus States: Different colors for primary/secondary focus
  • Quest Markers: Highlight quest-related objects
  • Accessibility: Help players identify interactive elements

How It Worksâ€‹

The component uses Unreal's Overlay Material system to render an outline effect:

  1. Finds a mesh component on the actor
  2. Creates a Dynamic Material Instance from the configured Overlay Material
  3. Applies/removes the overlay material at runtime
  4. Updates material parameters (color, thickness) dynamically

Setupâ€‹

1. Add Component to Actorâ€‹

Add ERPOutlineComponent to any actor you want to highlight:

Blueprint:

  • Open your actor Blueprint (door, item, enemy)
  • Click Add Component â†’ Search for ERPOutlineComponent

C++:

2. Configure Overlay Materialâ€‹

In the component details:

  • Overlay Material: Select your outline material (must be Unlit + Opaque)
  • Default Color: Initial outline color (default: cyan)
  • Default Thickness: Initial outline width (default: 1.0)

3. Create Overlay Materialâ€‹

Create a material that works with mesh overlays:

Material Setup:

  • Shading Model: Unlit
  • Blend Mode: Opaque
  • Parameters:
    • OutlineColor (Vector Parameter)
    • OutlineThickness (Scalar Parameter)

C++ Usageâ€‹

Enabling outline from C++:

void AMyPlayerController::OnCandidateAcquired(AActor* Candidate)
{
    if (auto* OutlineComp = Candidate->FindComponentByClass<UERPOutlineComponent>())
    {
        OutlineComp->EnableOutline();
        OutlineComp->SetOutlineColor(FLinearColor(1.0f, 0.5f, 0.0f)); // Orange
        OutlineComp->SetOutlineThickness(2.0f);
    }
}

void AMyPlayerController::OnCandidateLost(AActor* Candidate)
{
    if (auto* OutlineComp = Candidate->FindComponentByClass<UERPOutlineComponent>())
    {
        OutlineComp->DisableOutline();
    }
}

Best Practicesâ€‹

Performanceâ€‹

  • âœ… Do: Add component only to actors that need highlighting
  • âœ… Do: Disable outline when not visible to player
  • âœ… Do: Reuse the same material across all outlines
  • âŒ Don't: Create unique materials per actor
  • âŒ Don't: Update colors every frame unnecessarily

Visual Designâ€‹

  • âœ… Do: Use consistent colors for similar object types
  • âœ… Do: Use contrasting colors against environment
  • âœ… Do: Test outline visibility in different lighting
  • âŒ Don't: Use too many different outline colors
  • âŒ Don't: Make outlines so thick they obscure the object

Architectureâ€‹

  • âœ… Do: Control outlines from gameplay systems (perception, interaction)
  • âœ… Do: Check if component exists before using (optional component)
  • âŒ Don't: Put gameplay logic in the outline component
  • âŒ Don't: Use outlines as the only interaction indicator

Troubleshootingâ€‹

Outline not visible:

  • Verify the overlay material is set correctly
  • Check that the material is Unlit + Opaque
  • Ensure the mesh has a valid static/skeletal mesh
  • Enable outline explicitly with EnableOutline()

Outline looks wrong:

  • Check material parameters are exposed correctly
  • Verify parameter names match exactly: OutlineColor, OutlineThickness
  • Test the material in isolation first

Performance issues:

  • Reduce number of outlined actors visible at once
  • Simplify the overlay material
  • Use LODs appropriately
