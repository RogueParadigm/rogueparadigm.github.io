Renaming Changelog - ERPInteractionVisualFeedbackComponent → ERPBaseInteractionFeedbackComponent

Date: 2026-01-28​

Summary​

The interaction visual feedback component has been renamed to better reflect its purpose as a base class for Blueprint-driven implementations.

Changes​

1. C++ Files Renamed​

Before:

  • ERPInteractionVisualFeedbackComponent.h
  • ERPInteractionVisualFeedbackComponent.cpp

After:

  • ERPBaseInteractionFeedbackComponent.h
  • ERPBaseInteractionFeedbackComponent.cpp

Location: Plugins/ElysPerceptionPlugin/Source/ElysPerceptionPlugin/Public|Private/Domains/Interaction/

2. Class Renamed​

Before:

class UERPInteractionVisualFeedbackComponent : public UActorComponent

After:

class UERPBaseInteractionFeedbackComponent : public UActorComponent

3. Implementation Changes​

The component has been simplified to be 100% Blueprint-driven:

Removed:

  • OutlineComponent property
  • WorldSpaceWidgetComponent property
  • HandleFocusChanged() native implementation
  • HandleDescriptorUpdated() native implementation

Kept:

  • SetInteractionFocused(bool) - Triggers Blueprint event
  • UpdateInteractionDescriptor(const FERPInteractionDescriptor&) - Triggers Blueprint event
  • IsInteractionFocused() - Getter
  • Blueprint Events: OnInteractionFocused, OnDescriptorUpdated

4. Migration Path​

A ClassRedirect has been added to Config/DefaultEngine.ini:

[CoreRedirects]
+ClassRedirects=(OldName="ERPInteractionVisualFeedbackComponent",NewName="ERPBaseInteractionFeedbackComponent")

What this means:

  • All existing Blueprints using the old component will automatically use the new name
  • No manual Blueprint updates required
  • Recompilation will apply the redirect

5. Documentation Updated​

All documentation files have been updated:

  • Blueprint-Interaction-Quick-Reference.md
  • Blueprint-Templates.md
  • Blueprint-Interaction-Implementation-Guide.md
  • CHEAT-SHEET.md
  • FULL_DOCUMENTATION.md
  • IMPLEMENTATION_SUMMARY.md
  • Interaction-And-Targeting.md
  • START-HERE.md
  • VisualFeedbackComponent.md

Impact​

✅ No Breaking Changes​

Thanks to the ClassRedirect:

  • Existing Blueprints continue to work
  • No asset corruption
  • No manual fixes required

⚠️ Blueprint Logic Required​

The component no longer has native visual feedback behavior. Users must:

  1. Override OnInteractionFocused Blueprint event to control visuals (outline, widgets, etc.)
  2. Override OnDescriptorUpdated Blueprint event to update UI content

Example Blueprint Implementation​

In your Interactable Actor Blueprint:​

Event: OnInteractionFocused

OnInteractionFocused (bFocused)
  ↓
[Branch]
  Condition: bFocused ?
  TRUE →
    [WidgetComponent].SetVisibility(true)
    [OutlineComponent].EnableOutline()
  FALSE →
    [WidgetComponent].SetVisibility(false)
    [OutlineComponent].DisableOutline()

Event: OnDescriptorUpdated

OnDescriptorUpdated (Descriptor)
  ↓
[Get User Widget Object]
  Target: WidgetComponent
  ↓
[Cast to WBP_InteractionPrompt]
  ↓
[UpdateFromDescriptor]
  Descriptor: Descriptor

Next Steps​

  1. Recompile the plugin (Visual Studio or Unreal Editor)
  2. Reopen Unreal (if already open)
  3. Verify that your interactable actors still work
  4. Implement the Blueprint events if not already done
