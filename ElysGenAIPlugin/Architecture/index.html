Component Hierarchyâ€‹

Audio Flowâ€‹

Key Points:

  • Single audio capture point
  • Fan-out to multiple consumers
  • Consumer pattern for extensibility

STT Architectureâ€‹

Layers:

  1. Component - Blueprint API, event broadcasting
  2. Context - Per-actor state, audio buffering
  3. Subsystem - Backend lifecycle, context tracking
  4. Backend - Model loading, inference
  5. Model - Whisper.cpp C library

LLM Architectureâ€‹

Layers:

  1. Component - Blueprint API, conversation management
  2. Context - Message history, generation state
  3. Subsystem - Backend lifecycle, context tracking
  4. Backend - Model loading, text generation
  5. Model - llama.cpp with GGUF format

Interface Hierarchyâ€‹

Service Interfaceâ€‹

All backends implement IERP_GenAIService:

class IERP_GenAIService
{
    // Lifecycle
    virtual bool Initialize() = 0;
    virtual void Shutdown() = 0;
    virtual bool IsReady() const = 0;
    
    // State management
    virtual EElysGenAIServiceState GetProcessingState() const = 0;
    
    // Error handling
    virtual EElysGenAIErrorCode GetLastError() const = 0;
    virtual FString GetLastErrorMessage() const = 0;
    virtual void ClearError() = 0;
};

class IERP_STTBackend : public IERP_GenAIService

Subsystem Architectureâ€‹

GameInstanceSubsystem Patternâ€‹

All subsystems inherit from UGameInstanceSubsystem:

UCLASS()
class UERP_STTSubsystem : public UGameInstanceSubsystem
{
    GENERATED_BODY()
    
public:
    // Subsystem lifecycle (called by engine)
    virtual void Initialize(FSubsystemCollectionBase& Collection) override;
    virtual void Deinitialize() override;
    
    // Backend management
    TScriptInterface<IERP_STTBackend> GetBackend() const;
    
    // Context tracking
    void RegisterContext(FERP_STTContext* Context);
    void UnregisterContext(FERP_STTContext* Context);
    
private:
    TScriptInterface<IERP_STTBackend> Backend;
    TArray<FERP_STTContext*> ActiveContexts;
};

  • âœ… Singleton per GameInstance
  • âœ… Automatic lifecycle management
  • âœ… Survives level transitions
  • âœ… Accessible from any actor

Data Flowâ€‹

STT Processing Pipelineâ€‹

1. Microphone Input (OS Audio API)
    â†“
2. UERP_AudioCaptureSubsystem
    - Captures raw PCM audio
    - Converts to float32 [-1.0, 1.0]
    - Writes to ring buffer
    â†“
3. Fan-Out to Consumers
    - OnAudioDataReceived(FElysAudioBuffer)
    - Called on all registered IERP_AudioConsumer
    â†“
4. FERP_STTContext (implements IERP_AudioConsumer)
    - Receives audio buffer
    - Accumulates audio (e.g., 3 seconds)
    - Detects voice activity (VAD)
    â†“
5. UERP_STTSubsystem
    - Context requests transcription
    - Routes to backend
    â†“
6. UERP_WhisperBackend
    - Converts float32 to int16
    - Calls whisper.cpp API
    - Runs inference (worker thread)
    â†“
7. FElysSTTResult
    - TranscribedText: FString
    - Confidence: float
    - Language: FString
    - IsFinal: bool
    â†“
8. UERP_STTComponent
    - Receives result via delegate
    - Broadcasts OnTranscriptionComplete (Blueprint event)
    â†“
9. Game Logic (Blueprint/C++)
    - Handles transcription
    - Triggers game actions

LLM Generation Pipelineâ€‹

1. Game Logic (Blueprint/C++)
    - Calls UERP_LLMComponent::SendMessage()
    â†“
2. UERP_LLMComponent
    - Adds message to history
    - Forwards to context
    â†“
3. FERP_LLMContext
    - Builds prompt from history
    - Calls subsystem
    â†“
4. UERP_LLMSubsystem
    - Routes to backend
    â†“
5. UERP_LlamaCppBackend
    - Tokenizes prompt
    - Generates tokens (worker thread)
    - Optionally streams tokens
    â†“
6. FElysLLMResult
    - GeneratedText: FString
    - TokenCount: int32
    - FinishReason: enum
    â†“
7. UERP_LLMComponent
    - Receives result via delegate
    - Broadcasts OnGenerationComplete (Blueprint event)
    â†“
8. Game Logic (Blueprint/C++)
    - Displays NPC response
    - Triggers dialogue actions

Threading Modelâ€‹

Main Threadâ€‹

  • Component API calls
  • Blueprint event broadcasting
  • Subsystem management
  • Context state updates

Worker Threadsâ€‹

  • Audio capture (OS thread)
  • STT inference (Whisper.cpp)
  • LLM generation (llama.cpp)
  • Model loading/unloading

Thread Safetyâ€‹

  • Delegates fire on Game Thread - Safe for Blueprint
  • Async operations - Results marshaled to game thread
  • No shared mutable state - Contexts are independent

Memory Managementâ€‹

Model Lifecycleâ€‹

// Single shared model per subsystem
UERP_STTSubsystem::Initialize()
{
    Backend = CreateBackend();
    Backend->LoadModel(ModelPath); // ~74MB for Whisper base.en
}

UERP_STTSubsystem::Deinitialize()
{
    Backend->UnloadModel(); // Frees model memory
}

Context Lifecycleâ€‹

// Contexts owned by components via TUniquePtr
UERP_STTComponent::BeginPlay()
{
    Context = MakeUnique<FERP_STTContext>(this);
    Subsystem->RegisterContext(Context.Get());
}

UERP_STTComponent::EndPlay()
{
    Subsystem->UnregisterContext(Context.Get());
    Context.Reset(); // Automatic cleanup
}

Memory Profile:

  • Whisper base.en: ~74MB
  • Phi-3-mini Q4: ~2.7GB
  • Per-context overhead: ~10KB (negligible)

Configuration Systemâ€‹

Project Settingsâ€‹

All settings in UElysGenAISettings (UDeveloperSettings):

UCLASS(Config=Game, DefaultConfig, meta=(DisplayName="Elys GenAI Framework"))
class UElysGenAISettings : public UDeveloperSettings
{
    // Audio
    UPROPERTY(Config, EditAnywhere, Category="Audio")
    int32 DefaultSampleRate = 16000;
    
    // STT
    UPROPERTY(Config, EditAnywhere, Category="STT")
    EElysSTTBackendType STTBackend = EElysSTTBackendType::Whisper;
    
    UPROPERTY(Config, EditAnywhere, Category="STT")
    FString STTModelPath;
    
    // LLM
    UPROPERTY(Config, EditAnywhere, Category="LLM")
    EElysLLMBackendType LLMBackend = EElysLLMBackendType::LlamaCpp;
    
    UPROPERTY(Config, EditAnywhere, Category="LLM")
    FString LLMModelPath;
};

Access in code:

const UElysGenAISettings* Settings = GetDefault<UElysGenAISettings>();
int32 SampleRate = Settings->DefaultSampleRate;
