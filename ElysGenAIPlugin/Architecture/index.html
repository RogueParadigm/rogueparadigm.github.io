Architecture Overview

Quick reference guide for ElysGenAI modules - which component to use for what task.

Modules at a Glanceâ€‹

Module What It Does When To Use It
ElysGenAICore Provides shared types and interfaces Import for custom backend implementations
ElysGenAIAudio Captures microphone input and distributes to consumers Add to project for any voice/audio processing feature
ElysGenAISTT Converts speech to text using Whisper Use for voice commands, transcription, dialogue input
ElysGenAILLM Generates NPC dialogue using language models Use for conversational AI, dynamic responses

Key Componentsâ€‹

Audio Moduleâ€‹

  • UERP_AudioCaptureSubsystem - Singleton that captures microphone audio. Automatically starts on game launch.
  • IERP_AudioConsumer - Implement this interface to receive audio data (STT contexts use this internally).

STT Moduleâ€‹

  • UERP_STTComponent - Add to any actor that needs speech recognition. Broadcasts OnTranscriptionComplete events.
  • UERP_STTSubsystem - Manages the Whisper model lifecycle. Access via GetGameInstance()->GetSubsystem().

LLM Moduleâ€‹

  • UERP_LLMComponent - Add to NPCs for conversational AI. Call SendMessage() to get responses.
  • UERP_LLMSubsystem - Manages language model lifecycle. Loads model on initialization.

Quick Referenceâ€‹

Common Tasksâ€‹

To add voice commands to a player:

  1. Add UERP_STTComponent to player actor
  2. Bind to OnTranscriptionComplete event
  3. Process voice text in Blueprint/C++

To create an NPC with AI dialogue:

  1. Add UERP_LLMComponent to NPC actor
  2. Call SendMessage() with player input
  3. Bind to OnGenerationComplete for response

To configure settings:

  • Edit > Project Settings > Plugins > Elys GenAI Framework
  • Set model paths, sample rates, backend types
