Audio Capture System

Understanding ElysGenAI's audio capture, routing, and consumer pattern.

Overview‚Äã

UERP_AudioCaptureSubsystem captures microphone input and distributes it to multiple consumers (STT, voice chat, recording).

Key Features:

  • Single capture point
  • Fan-out to multiple consumers
  • Push-to-talk modes
  • Mute/unmute support
  • Ring buffer for audio history

Consumer Pattern‚Äã

Implementing a Consumer‚Äã

// YourComponent.h
UCLASS()
class UMyAudioConsumerERP : public UActorComponent, public IERP_AudioConsumer
{
    GENERATED_BODY()
    
public:
    // IERP_AudioConsumer interface
    virtual void OnAudioDataReceived_Implementation(const FElysAudioBuffer& Buffer) override
    {
        // Process audio
        ProcessAudio(Buffer.AudioData);
    }
    
    virtual FString GetConsumerName_Implementation() const override
    {
        return TEXT("MyAudioConsumer");
    }
};

Registering‚Äã

void UMyAudioConsumerERP::BeginPlay()
{
    Super::BeginPlay();
    
    UERP_AudioCaptureSubsystem* AudioSubsystem = 
        GetGameInstance()->GetSubsystem<UERP_AudioCaptureSubsystem>();
    
    if (AudioSubsystem)
    {
        AudioSubsystem->RegisterConsumer(this);
    }
}

void UMyAudioConsumerERP::EndPlay(const EEndPlayReason::Type Reason)
{
    if (UERP_AudioCaptureSubsystem* AudioSubsystem = 
        GetGameInstance()->GetSubsystem<UERP_AudioCaptureSubsystem>())
    {
        AudioSubsystem->UnregisterConsumer(this);
    }
    
    Super::EndPlay(Reason);
}

Push-to-Talk Modes‚Äã

Modes‚Äã

UENUM(BlueprintType)
enum class EElysPushToTalkMode : uint8
{
    AlwaysOn,      // Continuous capture
    PushToTalk,    // Hold key to talk
    PushToMute     // Hold key to mute
};

Configuration‚Äã

Blueprint:

Project Settings ‚Üí Elys GenAI Framework ‚Üí Audio
- PTT Mode: Push To Talk
- PTT Key: Left Shift

C++:

AudioSubsystem->SetPushToTalkMode(EElysPushToTalkMode::PushToTalk);
AudioSubsystem->SetPushToTalkActive(true); // Key pressed
AudioSubsystem->SetPushToTalkActive(false); // Key released

Input Binding‚Äã

// In PlayerController
void AMyPlayerController::SetupInputComponent()
{
    Super::SetupInputComponent();
    
    InputComponent->BindAction("VoiceChat", IE_Pressed, this, &AMyPlayerController::StartVoiceChat);
    InputComponent->BindAction("VoiceChat", IE_Released, this, &AMyPlayerController::StopVoiceChat);
}

void AMyPlayerController::StartVoiceChat()
{
    if (UERP_AudioCaptureSubsystem* AudioSubsystem = 
        GetGameInstance()->GetSubsystem<UERP_AudioCaptureSubsystem>())
    {
        AudioSubsystem->SetPushToTalkActive(true);
    }
}

void AMyPlayerController::StopVoiceChat()
{
    if (UERP_AudioCaptureSubsystem* AudioSubsystem = 
        GetGameInstance()->GetSubsystem<UERP_AudioCaptureSubsystem>())
    {
        AudioSubsystem->SetPushToTalkActive(false);
    }
}

Audio Format‚Äã

USTRUCT(BlueprintType)
struct FElysAudioFormat
{
    UPROPERTY(BlueprintReadWrite)
    int32 SampleRate = 16000;  // Hz
    
    UPROPERTY(BlueprintReadWrite)
    int32 NumChannels = 1;  // Mono
    
    UPROPERTY(BlueprintReadWrite)
    int32 BitsPerSample = 16;
};

Common Formats:

  • 16kHz, Mono, 16-bit - Whisper STT (recommended)
  • 48kHz, Stereo, 16-bit - Voice chat
  • 44.1kHz, Stereo, 16-bit - Recording

Examples‚Äã

Voice Recording‚Äã

UCLASS()
class UVoiceRecorderERP : public UActorComponent, public IERP_AudioConsumer
{
    GENERATED_BODY()
    
public:
    UFUNCTION(BlueprintCallable)
    void StartRecording();
    
    UFUNCTION(BlueprintCallable)
    void StopRecording();
    
    UFUNCTION(BlueprintCallable)
    void SaveToFile(const FString& Filename);
    
    virtual void OnAudioDataReceived_Implementation(const FElysAudioBuffer& Buffer) override
    {
        if (bIsRecording)
        {
            RecordedAudio.Append(Buffer.AudioData);
        }
    }
    
private:
    bool bIsRecording = false;
    TArray<float> RecordedAudio;
};

Audio Visualizer‚Äã

UCLASS()
class UAudioVisualizerERP : public UActorComponent, public IERP_AudioConsumer
{
    GENERATED_BODY()
    
public:
    virtual void OnAudioDataReceived_Implementation(const FElysAudioBuffer& Buffer) override
    {
        // Calculate RMS for volume bars
        float RMS = CalculateRMS(Buffer.AudioData);
        CurrentVolume = RMS;
        
        // Update frequency spectrum
        FFT(Buffer.AudioData, FrequencyBins);
        
        OnVisualizationUpdate.Broadcast(CurrentVolume, FrequencyBins);
    }
    
    UPROPERTY(BlueprintAssignable)
    FOnVisualizationUpdate OnVisualizationUpdate;
    
private:
    float CurrentVolume;
    TArray<float> FrequencyBins;
    
    float CalculateRMS(const TArray<float>& Audio);
    void FFT(const TArray<float>& Audio, TArray<float>& OutBins);
};

Troubleshooting‚Äã

No Audio Captured‚Äã

Check:

  1. Microphone permissions (OS settings)
  2. Default audio device in Windows/Mac settings
  3. IsCapturing() returns true
  4. Consumer registered with subsystem

Low Quality Audio‚Äã

Solutions:

  1. Increase sample rate (16kHz ‚Üí 48kHz)
  2. Reduce background noise
  3. Use better microphone
  4. Enable noise suppression (future feature)

Clicks/Pops in Audio‚Äã

Cause: Buffer underrun

Solution:

// Increase buffer duration
AudioSubsystem->SetBufferDuration(200); // ms (default: 100ms)
