Troubleshooting

Common issues and solutions for ElysGenAIPlugin.

Installation Issues‚Äã

Plugin Not Appearing in Editor‚Äã

Symptom: ElysGenAI not listed in Plugins window

Solutions:

  1. Verify .uplugin file exists: 
    YourProject/Plugins/ElysGenAIPlugin/ElysGenAIPlugin.uplugin
  2. Check .uplugin syntax (valid JSON)
  3. Regenerate project files:
    • Right-click .uproject ‚Üí Generate Visual Studio Project Files
  4. Delete Intermediate, Binaries, Saved folders and regenerate
  5. Check Output Log for plugin loading errors

Compilation Errors‚Äã

Error: unresolved external symbol UDeveloperSettings

Solution: Add to game .Build.cs:

PublicDependencyModuleNames.AddRange(new string[] {
    "DeveloperSettings"
});

Error: Module 'ElysGenAICore' not found

Solution:

  1. Check plugin is enabled in .uproject
  2. Verify all module .Build.cs files exist
  3. Clean and rebuild project

Error: Cannot open include file: 'IERP_STTBackend.h'

Solution:

  1. Verify file exists in ElysGenAICore/Public/
  2. Check module dependency in .Build.cs: 
    PublicDependencyModuleNames.Add("ElysGenAICore");

Audio Capture Issues‚Äã

No Audio Captured‚Äã

Symptoms:

  • IsCapturing() returns false
  • No audio data in consumers
  • Silence in recordings

Solutions:

  1. Check Microphone Permissions

    • Windows: Settings ‚Üí Privacy ‚Üí Microphone ‚Üí Allow desktop apps
    • Mac: System Preferences ‚Üí Security & Privacy ‚Üí Microphone ‚Üí Grant access

  2. Verify Default Audio Device

    • Windows: Sound Settings ‚Üí Input ‚Üí Select correct microphone
    • Mac: System Preferences ‚Üí Sound ‚Üí Input ‚Üí Select microphone

  3. Check Subsystem Initialization

    UERP_AudioCaptureSubsystem* AudioSubsystem = 
        GetGameInstance()->GetSubsystem<UERP_AudioCaptureSubsystem>();

    if (!AudioSubsystem)
    {
        UE_LOG(LogTemp, Error, TEXT("Audio subsystem not found!"));
    }

    if (!AudioSubsystem->IsCapturing())
    {
        AudioSubsystem->StartCapture();
    }

  4. Check Output Log Look for:

    LogElysAudio: Audio capture failed to initialize
    LogElysAudio: No audio input device available

Low Audio Quality‚Äã

Symptoms:

  • Distorted audio
  • Clicks/pops
  • Static noise

Solutions:

  1. Increase Sample Rate

    // Project Settings ‚Üí Audio ‚Üí Sample Rate
    Settings->DefaultSampleRate = 48000;  // Default: 16000

  2. Increase Buffer Size

    // Larger buffer = less clicks, but more latency
    AudioSubsystem->SetBufferDuration(200);  // Default: 100ms

  3. Reduce Background Noise

    • Move away from fans, AC, etc.
    • Use noise-canceling microphone
    • Enable VAD (future feature)

  4. Check Microphone Quality

    • Test in other applications (Discord, Zoom)
    • Try different USB port
    • Update audio drivers

STT Issues‚Äã

No Transcriptions‚Äã

Symptoms:

  • OnTranscriptionComplete never fires
  • Empty transcription results

Solutions:

  1. Check Listening State

    if (!STTComponent->IsListening())
    {
        STTComponent->StartListening();
    }

  2. Check Model Loaded

    UERP_STTSubsystem* STTSubsystem = 
        GetGameInstance()->GetSubsystem<UERP_STTSubsystem>();

    if (!STTSubsystem->GetBackend()->IsModelLoaded())
    {
        UE_LOG(LogTemp, Error, TEXT("STT model not loaded!"));
    }

  3. Verify Model Path

    • Check Project Settings ‚Üí STT ‚Üí Model Path
    • Default: Plugin bundled model
    • If custom: verify file exists and is valid Whisper model

  4. Check Network Mode

    // STT only works on client
    if (GetNetMode() == NM_DedicatedServer)
    {
        UE_LOG(LogTemp, Warning, TEXT("STT disabled on dedicated server"));
    }

  5. Speak Louder/Clearer

    • Enunciate clearly
    • Speak directly into microphone
    • Reduce background noise

Low Confidence Scores‚Äã

Symptom: Confidence always < 0.5

Solutions:

  1. Adjust Minimum Confidence

    STTComponent->SetMinConfidence(0.3f);  // Default: 0.5

  2. Use Larger Model

    // Project Settings ‚Üí STT ‚Üí Model Path
    // Change: base.en ‚Üí small.en or medium.en

  3. Enable VAD

    STTComponent->SetEnableVAD(true);

  4. Improve Audio Quality

    • Better microphone
    • Quieter environment
    • Higher sample rate

Wrong Language Detected‚Äã

Symptom: Transcription in wrong language

Solutions:

  1. Set Language Explicitly

    STTComponent->SetLanguageCode(TEXT("en"));  // English

  2. Use Language-Specific Model

    • English: base.en, small.en
    • Multilingual: base, small, medium

  3. Check Model Supports Language

    • Some models are English-only
    • Verify language support: https://github.com/openai/whisper

LLM Issues‚Äã

Model Not Loading‚Äã

Symptoms:

  • IsModelLoaded() returns false
  • Generation fails immediately
  • "Model not found" errors

Solutions:

  1. Check Model Path

    // Project Settings ‚Üí LLM ‚Üí Model Path
    // Default: Content/AI/Models/Phi3/phi-3-mini-4k-instruct-q4.gguf

  2. Verify Model Format

    • Must be GGUF format
    • Check file extension: .gguf
    • Verify not corrupted (check file size)

  3. Check File Permissions

    • Read access required
    • Not locked by another process

  4. Check Available RAM

    • Phi-3-mini Q4: ~2.7GB required
    • Close other applications if low on RAM

Slow Generation‚Äã

Symptom: Very slow token generation (>1 second per token)

Solutions:

  1. Reduce Context Length

    // Project Settings ‚Üí LLM ‚Üí Context Length
    Settings->LLMContextLength = 2048;  // Default: 4096

  2. Use Fewer Threads

    // Counterintuitive but can help if CPU-bound
    Settings->LLMNumThreads = 2;  // Default: 4

  3. Reduce Max Tokens

    LLMComponent->SetMaxTokens(50);  // Default: 256

  4. Use Smaller Model

    • Phi-3-mini ‚Üí TinyLlama
    • Check model size vs. generation speed

  5. Close Other Applications

    • Free up CPU resources
    • Disable background processes

Out of Memory‚Äã

Error: Failed to allocate memory for model

Solutions:

  1. Use Smaller Model

    • Phi-3-mini Q4: 2.7GB
    • TinyLlama Q4: 800MB

  2. Increase Virtual Memory

    • Windows: Settings ‚Üí System ‚Üí About ‚Üí Advanced System Settings ‚Üí Performance ‚Üí Advanced ‚Üí Virtual Memory
    • Set larger page file

  3. Close Other Applications

    • Free up RAM
    • Check Task Manager for memory hogs

  4. Use Lower Quantization

    • Q4 (current): 4-bit quantization
    • Q8 (larger but better): 8-bit quantization
    • Trade quality for memory

Inconsistent Responses‚Äã

Symptom: NPC gives random/inconsistent answers

Solutions:

  1. Lower Temperature

    LLMComponent->SetTemperature(0.3f);  // Default: 0.7

  2. Improve System Prompt

    LLMComponent->SetSystemPrompt(
        TEXT("You are Gareth, a merchant. "
             "You only sell weapons and armor. "
             "Stay in character. Be consistent."));

  3. Provide Examples

    // Few-shot learning
    SetSystemPrompt(
        TEXT("Examples:\n"
             "Player: What do you sell?\n"
             "Merchant: Weapons and armor!\n\n"
             "Player: Do you have potions?\n"
             "Merchant: No, I only sell weapons and armor."));

  4. Clear History Regularly

    LLMComponent->ClearHistory();

Performance Issues‚Äã

High CPU Usage‚Äã

Symptom: Game stutters, low FPS during AI processing

Solutions:

  1. Use Async Processing

    • STT and LLM automatically run on worker threads
    • Verify not blocking game thread

  2. Reduce Thread Count

    Settings->STTNumThreads = 2;
    Settings->LLMNumThreads = 2;

  3. Use Smaller Models

    • STT: tiny/base instead of small/medium
    • LLM: TinyLlama instead of Phi-3

  4. Limit Active Contexts

    • Don't create 100+ STT contexts simultaneously
    • Pool and reuse contexts

High Memory Usage‚Äã

Symptom: RAM usage over 4GB

Solutions:

  1. Single Model Instance

    • Subsystems ensure single shared model
    • Don't create multiple backends

  2. Clear Context History

    LLMComponent->ClearHistory();  // Frees token history

  3. Limit Context Window

    Settings->LLMContextLength = 2048;  // Reduces memory per context

  4. Use Quantized Models

    • Q4 models use 4x less memory than full precision
    • Bundled models already quantized

Multiplayer Issues‚Äã

Audio Captured on Server‚Äã

Symptom: Dedicated server trying to capture audio

Solution:

void UERP_STTComponent::BeginPlay()
{
    Super::BeginPlay();
    
    // ALWAYS check network mode
    if (GetNetMode() == NM_DedicatedServer)
    {
        return;  // Skip audio capture on server
    }
    
    StartListening();
}

RPC Validation Failing‚Äã

Error: ServerVoiceCommand_Validation failed

Solutions:

  1. Check Parameter Limits

    bool AMyPC::ServerVoiceCommand_Validation(const FString& Command)
    {
        // Too strict?
        return Command.Len() > 0 && Command.Len() < 256;
    }

  2. Add Logging

    bool AMyPC::ServerVoiceCommand_Validation(const FString& Command)
    {
        if (Command.Len() > 256)
        {
            UE_LOG(LogTemp, Warning, TEXT("Command too long: %d"), Command.Len());
            return false;
        }
        return true;
    }

Blueprint Issues‚Äã

Component Not Appearing‚Äã

Symptom: Can't find ERP_STTComponent in Add Component menu

Solutions:

  1. Check Plugin Enabled

    • Edit ‚Üí Plugins ‚Üí Search "ElysGenAI" ‚Üí Enabled

  2. Restart Editor

    • Plugin changes require restart

  3. Check Class Specifiers

    UCLASS(BlueprintType, ClassGroup=(ElysGenAI), 
           meta=(BlueprintSpawnableComponent))

Events Not Firing‚Äã

Symptom: OnTranscriptionComplete never triggers in Blueprint

Solutions:

  1. Verify Event Bound

    • Component Details ‚Üí Events ‚Üí Check event created

  2. Check Delegate Type

    UPROPERTY(BlueprintAssignable, Category="ElysGenAI|STT")
    FElysSTTResultDelegate OnTranscriptionComplete;

  3. Test in C++

    STTComponent->OnTranscriptionComplete.AddDynamic(
        this, &AMyActor::OnTranscription);
