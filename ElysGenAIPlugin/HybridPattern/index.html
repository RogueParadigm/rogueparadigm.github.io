Hybrid Pattern Explained

Understanding why ElysGenAI uses plain C++ structs + UObject component wrappers.

The Problem‚Äã

Traditional Unreal Engine approaches have tradeoffs:

Approach 1: Pure UObject‚Äã

UCLASS()
class UERP_STTContext : public UObject
{
    // Everything is a UObject
    TArray<float> AudioBuffer;     // ‚ùå UObject overhead
    FElysSTTResult LastResult;     // ‚ùå Garbage collection
    // ... heavy processing logic  // ‚ùå Not ideal for hot paths
};

Downsides:

  • ‚ùå UObject overhead (~200 bytes per instance)
  • ‚ùå Garbage collection pressure
  • ‚ùå Slower allocation/deallocation
  • ‚ùå Not cache-friendly for tight loops

Approach 2: Pure C++ Struct‚Äã

struct FElysSTTContext
{
    // Plain C++ struct
    TArray<float> AudioBuffer;     // ‚úÖ No overhead
    FElysSTTResult LastResult;     // ‚úÖ Stack/heap efficient
    // ... heavy processing logic  // ‚úÖ Fast
};

Downsides:

  • ‚ùå No Blueprint exposure
  • ‚ùå Manual lifetime management
  • ‚ùå No UPROPERTY reflection
  • ‚ùå Can't use delegates easily

The Solution: Hybrid Pattern‚Äã

Use both - each for what it's good at!

Plain C++ Struct (Internal Logic)‚Äã

// Heavy lifting - lightweight, fast, cache-friendly
struct FERP_STTContext
{
    // State
    TArray<float> AudioBuffer;
    FElysSTTResult LastResult;
    bool bIsListening;
    
    // Logic
    void AccumulateAudio(const TArray<float>& NewAudio);
    void ProcessAudio();
    void RequestTranscription();
    
    // Delegates (C++ only)
    DECLARE_MULTICAST_DELEGATE_OneParam(FOnTranscription, const FElysSTTResult&);
    FOnTranscription OnTranscriptionResult;
};

Characteristics:

  • ‚úÖ No UObject overhead
  • ‚úÖ Fast allocation/deallocation
  • ‚úÖ Cache-friendly
  • ‚úÖ Ideal for tight loops
  • ‚úÖ Can use C++ delegates

UObject Component (Blueprint API)‚Äã

// Blueprint wrapper - exposes only what's needed
UCLASS(BlueprintType, ClassGroup=(ElysGenAI), meta=(BlueprintSpawnableComponent))
class UERP_STTComponent : public UActorComponent
{
    GENERATED_BODY()
    
public:
    // Blueprint API
    UFUNCTION(BlueprintCallable, Category="ElysGenAI|STT")
    void StartListening();
    
    UFUNCTION(BlueprintCallable, Category="ElysGenAI|STT")
    void StopListening();
    
    UPROPERTY(BlueprintReadOnly, Category="ElysGenAI|STT")
    bool bIsListening;
    
    // Blueprint event
    UPROPERTY(BlueprintAssignable, Category="ElysGenAI|STT")
    FElysSTTResultDelegate OnTranscriptionComplete;
    
private:
    // Owns the lightweight context
    TUniquePtr<FERP_STTContext> Context;
    
    // Forwards context events to Blueprint
    void HandleContextTranscription(const FElysSTTResult& Result);
};

Characteristics:

  • ‚úÖ Blueprint-friendly
  • ‚úÖ UPROPERTY/UFUNCTION exposure
  • ‚úÖ Dynamic multicast delegates
  • ‚úÖ Automatic cleanup
  • ‚úÖ Actor component integration

How They Work Together‚Äã

Ownership‚Äã

UERP_STTComponent::BeginPlay()
{
    // Component creates and owns context
    Context = MakeUnique<FERP_STTContext>(this);
    
    // Bind C++ delegate to forward events
    Context->OnTranscriptionResult.AddRaw(
        this, &UERP_STTComponent::HandleContextTranscription
    );
    
    // Register with subsystem
    Subsystem->RegisterContext(Context.Get());
}

UERP_STTComponent::EndPlay()
{
    // Automatic cleanup via TUniquePtr
    Subsystem->UnregisterContext(Context.Get());
    Context.Reset(); // Destructor called here
}

Method Forwarding‚Äã

// Blueprint calls component method
void UERP_STTComponent::StartListening()
{
    if (Context)
    {
        Context->StartListening(); // Forward to context
        bIsListening = Context->IsListening(); // Sync state
    }
}

// Context processes audio, fires C++ delegate
void FERP_STTContext::OnTranscriptionReady(const FElysSTTResult& Result)
{
    // Fire C++ multicast delegate
    OnTranscriptionResult.Broadcast(Result);
}

// Component receives C++ event, broadcasts to Blueprint
void UERP_STTComponent::HandleContextTranscription(const FElysSTTResult& Result)
{
    // Fire Blueprint dynamic multicast delegate
    OnTranscriptionComplete.Broadcast(Result);
}

Data Flow‚Äã

Blueprint
    ‚Üì (UFUNCTION call)
UERP_STTComponent
    ‚Üì (method forward)
FERP_STTContext
    ‚Üì (processing)
[Heavy Logic: Audio buffering, VAD, inference requests]
    ‚Üì (C++ delegate)
FERP_STTContext::OnTranscriptionResult
    ‚Üì (forward)
UERP_STTComponent::HandleContextTranscription
    ‚Üì (Blueprint delegate)
UERP_STTComponent::OnTranscriptionComplete
    ‚Üì (Blueprint event)
Game Logic

Benefits‚Äã

Performance‚Äã

Memory:

sizeof(FERP_STTContext) ‚âà 50-100 bytes (audio buffer + state)
sizeof(UERP_STTComponent) ‚âà 200 bytes (UObject overhead)

vs Pure UObject approach:

sizeof(UERP_STTContext) ‚âà 200 bytes + UObject overhead per instance

CPU:

  • Context processing happens in tight C++ loops
  • No UObject virtual function overhead
  • Cache-friendly for audio processing
  • Component only touched for Blueprint communication

Scalability‚Äã

Multiple Contexts:

// 100 actors with STT components
for (int i = 0; i < 100; i++)
{
    // Lightweight contexts, minimal overhead
    FERP_STTContext* Context = new FERP_STTContext();
    // Only ~5-10KB per context
}

vs Pure UObject:

// 100 UObject contexts = significant GC pressure

Flexibility‚Äã

Easy to extend:

// Add new logic to context - no Blueprint recompilation
struct FERP_STTContext
{
    // New feature
    void EnableNoiseReduction(bool bEnable);
    float ApplyNoiseReduction(const TArray<float>& Audio);
};

// Expose to Blueprint only if needed
class UERP_STTComponent
{
    UFUNCTION(BlueprintCallable)
    void EnableNoiseReduction(bool bEnable)
    {
        Context->EnableNoiseReduction(bEnable);
    }
};

Pattern in Action‚Äã

STT Example‚Äã

// Context (lightweight, fast)
struct FERP_STTContext
{
    // State
    TArray<float> AudioBuffer;
    TArray<float> ProcessedAudio;
    int32 SampleRate;
    bool bIsListening;
    
    // Heavy processing
    void AccumulateAudio(const TArray<float>& NewAudio)
    {
        // Tight loop - no UObject overhead
        AudioBuffer.Append(NewAudio);
        
        if (AudioBuffer.Num() >= RequiredSamples)
        {
            ProcessBuffer();
        }
    }
    
    void ProcessBuffer()
    {
        // Apply VAD, normalization, etc.
        ProcessedAudio = ApplyVoiceActivityDetection(AudioBuffer);
        
        if (HasVoiceActivity(ProcessedAudio))
        {
            RequestTranscription();
        }
    }
    
    // C++ delegate
    FOnTranscription OnTranscriptionResult;
};

// Component (Blueprint wrapper)
UCLASS()
class UERP_STTComponent : public UActorComponent
{
    // Blueprint API
    UFUNCTION(BlueprintCallable)
    void StartListening()
    {
        Context->StartListening();
        bIsListening = true;
    }
    
    UPROPERTY(BlueprintAssignable)
    FElysSTTResultDelegate OnTranscriptionComplete;
    
    // Owns context
    TUniquePtr<FERP_STTContext> Context;
    
    // Forwards events
    void HandleTranscription(const FElysSTTResult& Result)
    {
        OnTranscriptionComplete.Broadcast(Result);
    }
};

LLM Example‚Äã

// Context (conversation state)
struct FERP_LLMContext
{
    // State
    TArray<FElysLLMMessage> MessageHistory;
    FString SystemPrompt;
    int32 TokenCount;
    
    // Logic
    FString BuildPrompt() const
    {
        FString Prompt = SystemPrompt + "\n";
        for (const auto& Msg : MessageHistory)
        {
            Prompt += FString::Printf(TEXT("%s: %s\n"), 
                *Msg.Role, *Msg.Content);
        }
        return Prompt;
    }
    
    void AddMessage(const FElysLLMMessage& Message)
    {
        MessageHistory.Add(Message);
        TokenCount += EstimateTokens(Message.Content);
        
        // Trim if over context limit
        if (TokenCount > MaxContextTokens)
        {
            TrimOldMessages();
        }
    }
    
    // C++ delegates
    FOnGeneration OnGenerationComplete;
    FOnToken OnTokenGenerated;
};

// Component (Blueprint wrapper)
UCLASS()
class UERP_LLMComponent : public UActorComponent
{
    // Blueprint API
    UFUNCTION(BlueprintCallable)
    void SendMessage(const FString& Message)
    {
        FElysLLMMessage Msg;
        Msg.Role = TEXT("user");
        Msg.Content = Message;
        Context->AddMessage(Msg);
        Context->RequestGeneration();
    }
    
    UPROPERTY(BlueprintAssignable)
    FElysLLMResultDelegate OnGenerationComplete;
    
    // Owns context
    TUniquePtr<FERP_LLMContext> Context;
};

vs Actor Components Only‚Äã

AspectHybrid PatternPure Components
Performance‚úÖ Fast (C++ struct)‚ùå Slower (UObject)
Memory‚úÖ Minimal overhead‚ùå UObject overhead
Blueprint‚úÖ Exposed via component‚úÖ Native
Scalability‚úÖ 1000s of contexts‚ö†Ô∏è GC pressure
Complexity‚ö†Ô∏è Two layers‚úÖ Single layer

vs Subsystems Only‚Äã

AspectHybrid PatternPure Subsystems
Per-Actor State‚úÖ Context per actor‚ùå Global state
Blueprint‚úÖ Component API‚ö†Ô∏è Static functions
Cleanup‚úÖ Automatic‚ö†Ô∏è Manual tracking
Scalability‚úÖ Independent contexts‚ö†Ô∏è Shared state

vs Data-Only Structs‚Äã

AspectHybrid PatternPure Structs
Blueprint‚úÖ Component wrapper‚ùå Not exposed
Logic‚úÖ In context‚ö†Ô∏è Elsewhere
Encapsulation‚úÖ Clear ownership‚ö†Ô∏è External logic
Events‚úÖ Delegates‚ùå Manual callbacks

When to Use Hybrid Pattern‚Äã

‚úÖ Use Hybrid When:‚Äã

  • Heavy processing per-actor (audio, inference)
  • Need Blueprint exposure
  • Want performance-critical loops
  • Multiple instances (100s-1000s)
  • Clear ownership semantics

‚ùå Don't Use Hybrid When:‚Äã

  • Simple data holders
  • Rarely updated state
  • No performance concerns
  • Single global instance
  • Blueprint-only logic

Implementation Guidelines‚Äã

1. Context Lifetime‚Äã

// Component manages context lifetime
class UERP_STTComponent
{
    void BeginPlay() override
    {
        Context = MakeUnique<FERP_STTContext>(this);
        Subsystem->RegisterContext(Context.Get());
    }
    
    void EndPlay(const EEndPlayReason::Type Reason) override
    {
        Subsystem->UnregisterContext(Context.Get());
        Context.Reset(); // Automatic cleanup
    }
    
private:
    TUniquePtr<FERP_STTContext> Context;
};

2. Event Forwarding‚Äã

// Bind C++ delegate in component constructor
UERP_STTComponent::BeginPlay()
{
    Context->OnTranscriptionResult.AddUObject(
        this, &UERP_STTComponent::HandleTranscription
    );
}

void UERP_STTComponent::HandleTranscription(const FElysSTTResult& Result)
{
    // Forward to Blueprint dynamic delegate
    OnTranscriptionComplete.Broadcast(Result);
}

3. State Synchronization‚Äã

// Sync component properties from context
void UERP_STTComponent::Tick(float DeltaTime)
{
    if (Context)
    {
        bIsListening = Context->IsListening();
        LastConfidence = Context->GetLastConfidence();
    }
}

Advanced: Custom Contexts‚Äã

You can create custom contexts for specialized use cases:

// Custom context for music recognition
struct FERP_MusicRecognitionContext
{
    TArray<float> MusicBuffer;
    TArray<FString> DetectedGenres;
    
    void AnalyzeMusic(const TArray<float>& Audio);
    void DetectBPM();
    void RecognizeMelody();
    
    FOnMusicDetected OnMusicDetected;
};

// Wrapper component
UCLASS()
class UERP_MusicRecognitionComponent : public UActorComponent
{
    UFUNCTION(BlueprintCallable)
    void StartListening();
    
    UPROPERTY(BlueprintAssignable)
    FElysMusicDetectedDelegate OnMusicDetected;
    
    TUniquePtr<FERP_MusicRecognitionContext> Context;
};

The hybrid pattern provides:

  • ‚ö° Performance - Fast C++ for processing
  • üéÆ Blueprint-Friendly - UObject wrapper for game logic
  • üì¶ Clean Architecture - Clear separation of concerns
  • üîß Extensible - Easy to add new features
  • üíæ Memory Efficient - Minimal overhead per instance

Best of both worlds!
