Tips for Each Patternâ€‹

  • Boss Fight: Use priority 15+ for critical music
  • Stingers: Keep short (<3 seconds) and punchy
  • Configs: Reuse setups across multiple levels
  • Day/Night: Use smooth fade transitions (5+ seconds)
  • Volume: Provide user control via settings
  • Debugging: Use Get Active Layer Names during development

Tips for Quartz Patternsâ€‹

  • Bar Quantization: Best for major transitions (combat, boss phases)
  • Beat Quantization: Best for quick effects (stingers, SFX)
  • OnBeat Events: Use sparingly - fires every beat (performance)
  • OnBar Events: Better for gameplay logic (less frequent)
  • BPM Changes: Smooth transitions (2-3 seconds) for tempo shifts
  • Timing Windows: 0.2 seconds is good for rhythm game "perfect" timing
  • VFX Sync: Spawn effects on beat 1 for strongest impact
  • Boss Patterns: Use bar multiples (4, 8, 16) for predictable rhythm
  • UI Animations: Scale animation speed by (CurrentBPM / 120)
  • Quantize Stingers: Use Bar for important stingers, Beat for quick ones

When to Use Quartz vs Immediateâ€‹

Use Immediate (Push Music Layer)â€‹

  • Non-musical games
  • Background ambiance
  • Menu music
  • First level music
  • Quick testing

Use Quantized (Push Music Layer Quantized)â€‹

  • Beat-driven games
  • Combat transitions
  • Boss music
  • Rhythm mechanics
  • Professional polish

Pro Tip: You can mix! Use immediate for base layers, quantized for transitions.
