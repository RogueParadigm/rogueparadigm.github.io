Migration Guide - From Old System to New ERP Music Engine

What Changed?‚Äã

Your old persistent music system (using ERP_MusicStateInterface in GameState) is now replaced by the new layer-based system.

Old System (What You Had)‚Äã

GameState (implements ERP_MusicStateInterface)
  - Playing Music: AudioComponent
  - Playing Music Name: Name

Custom Event: Start Persistent Music
  - Get Current Music (interface call)
  - Check if same track playing
  - FadeOut old, SpawnSound2D new
  - Set Current Music (interface call)

Custom Event: Stop Persistent Music
  - Get Current Music
  - FadeOut

New System (What to Use Now)‚Äã

‚úÖ Delete Old Blueprints‚Äã

Remove from Content:

  • ERP_BPF_MusicHelper.uasset (old)
  • ERP_MusicManager.uasset (old)
  • ERP_MusicStateInterface.uasset (old - now C++ interface)
  • ERP_Music_Structure.uasset (old - now C++ struct)

Keep:

  • ERP_MSS_MusicPlayer.uasset (MetaSound - still useful)
  • MSS_LoopingMusic.uasset (MetaSound - still useful)

‚úÖ Remove GameState Interface Implementation‚Äã

In your GameState Blueprint:

  1. Class Settings ‚Üí Interfaces
  2. Remove ERP_MusicStateInterface
  3. Delete GetCurrentMusic / SetCurrentMusic functions
  4. Delete variables:
    • Playing Music
    • Playing Music Name

Migration Examples‚Äã

Example 1: Start Persistent Music‚Äã

OLD:

Event Start Persistent Music
  Args: Music Name, Source Music, Volume
  ‚Üì
Get Current Music (interface)
  ‚Üì
Is Valid check
  ‚Üì
If (not same track):
    FadeOut current
    SpawnSound2D new
    FadeIn
    Set Current Music (interface)

NEW (Simple):

Event Start Persistent Music
  Args: Music Name, Source Music, Volume
  ‚Üì
Push Music Layer
  - Layer Name: Music Name
  - Music: Source Music
  - Priority: 0
  - Layer Mode: Replace
  - Volume: Volume
  - Persist Across Levels: true
  - Fade In: 1.0
  - Fade Out: (fade value)

NEW (Better - check if already playing):

Event Start Persistent Music
  Args: Music Name, Source Music, Volume
  ‚Üì
Branch: Is Layer Active (Music Name)?
  ‚Üì FALSE:
      Push Music Layer (as above)
  ‚Üì TRUE:
      (do nothing - already playing)

Example 2: Stop Persistent Music‚Äã

OLD:

Event Stop Persistent Music
  ‚Üì
Get Current Music (interface)
  ‚Üì
Is Valid
  ‚Üì
Set Current Music (clear)
FadeOut audio component

NEW:

Event Stop Persistent Music
  ‚Üì
Clear All Music Layers

Or if you want to stop just one layer:

Pop Music Layer
  - Layer Name: "YourMusicName"

  3. Configure:

Key Differences‚Äã

Old SystemNew System
Only 1 music at a timeMultiple layers possible
Manual track checkingAuto-managed by priority
Interface callsDirect function calls
Manual spawningAuto audio component pooling
Manual persistenceBuilt-in persistence flag
No prioritiesPriority system (0-100)
Manual state trackingAuto-tracked in subsystem

Benefits of New System‚Äã

‚úÖ Layers: Mix exploration + tension simultaneously
‚úÖ Priorities: Combat auto-overrides exploration
‚úÖ Auto-cleanup: Pop layer = auto-return to previous
‚úÖ No interface: Direct function calls
‚úÖ Music zones: Drop actors, no Blueprint needed
‚úÖ Stingers: Built-in support
‚úÖ Ducking: Auto-duck for dialogues
‚úÖ Configs: Reusable presets

Migration Checklist‚Äã

In GameState‚Äã

  • Remove ERP_MusicStateInterface implementation
  • Delete Playing Music variable
  • Delete Playing Music Name variable
  • Delete GetCurrentMusic function
  • Delete SetCurrentMusic function

In Content Browser‚Äã

  • Delete old ERP_BPF_MusicHelper Blueprint
  • Delete old ERP_MusicManager Blueprint
  • Delete old ERP_MusicStateInterface Blueprint
  • Delete old ERP_Music_Structure Blueprint
  • Keep MetaSound assets (still useful)

Update Custom Events‚Äã

  • Replace Start Persistent Music logic with Push Music Layer
  • Replace Stop Persistent Music logic with Pop Music Layer
  • Add priority values where combat/zones need override
  • Set Persist Across Levels: true where needed

Optional: Use New Features‚Äã

  • Replace zone trigger Blueprints with ERP_MusicZone actors
  • Add stingers for achievements/events
  • Add dialogue ducking
  • Create reusable music configs

Testing After Migration‚Äã

  1. Play in Editor

  2. Test each scenario:

    • Start music ‚Üí Should play
    • Combat starts ‚Üí Should switch
    • Combat ends ‚Üí Should return to exploration
    • Change level ‚Üí Music persists
    • Enter zone ‚Üí Zone music plays
    • Exit zone ‚Üí Previous music returns

  3. Check console for logs:

    • Open console (~)
    • Type: music.debug
    • Should show active layers

Common Issues‚Äã

"Music doesn't persist across levels"

  • Set Persist Across Levels: true when pushing layer

"Combat music doesn't override"

  • Set higher priority (e.g., Combat = 10, Exploration = 0)

"Old music still playing with new"

  • Use Layer Mode: Replace (not Additive)

"Music abruptly stops"

  • Check fade times (Fade Out time)

"Zone music doesn't trigger"

  • Verify ERP_MusicZone actor collision is enabled
  • Check Required Actor Tag if set
