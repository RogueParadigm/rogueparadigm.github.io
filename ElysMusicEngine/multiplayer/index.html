Network Frequencyâ€‹

PlayContext updates are optimized:

Advanced: Custom Replicationâ€‹

If you need server-authoritative music (rare), create a replicated actor:

UCLASS(Replicated)

{
    GENERATED_BODY()

    // Replicated music state
    UPROPERTY(ReplicatedUsing=OnRep_CurrentMusic)
    FName CurrentMusicLayer;

    // Called on clients when music changes
    UFUNCTION()
    void OnRep_CurrentMusic()
    {
        UERP_MusicSubsystem* MusicSys = GetGameInstance()->GetSubsystem<UERP_MusicSubsystem>();
        if (MusicSys && !CurrentMusicLayer.IsNone())
        {
            // Apply replicated music state
            MusicSys->PushMusicLayerByName(CurrentMusicLayer);
        }
    }

    // Server function to change music for everyone
    UFUNCTION(BlueprintCallable, BlueprintAuthorityOnly)
    void ServerSetMusic(FName LayerName)
    {
        CurrentMusicLayer = LayerName;
        // Replication automatically updates all clients
    }
};

When to use this:

  • Rhythm games requiring perfect sync
  • Server-controlled boss music phases
  • Turn-based games with music tied to game state

When NOT to use:

  • 99% of normal games - default client-side is better
